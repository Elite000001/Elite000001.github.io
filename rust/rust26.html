Part 3 of Build A Simple File Explorer in Rust

6 steps

In this part 3 of the series, you will extend the libfile-explorer crate and write a JSON API using axum, tokio, and serde. These crates help you write less boilerplate code while introducing async primitives.

Once done, you will modify the overall project as a cargo workspace. This allows you to run cargo commands at the root directory of the project as well as adding a new Rust project that runs the API server.

Ensure that you have finished Tutorial 1 and Tutorial 2 of this tutorial series.

    use std::fs;
use std::io;
use std::os::unix::fs::MetadataExt;
use std::path::Path;
use std::path::PathBuf;
use std::time::SystemTime;

use libroast::utils::copy_dir_all;
use serde::Deserialize;
use serde::Serialize;

/// A representation of what a filesystem has. For now, we have two kinds, it is either
/// a `Directory` or a `File`.
#[derive(Clone, Debug, Serialize, Deserialize)]
pub enum FsKind {
    Directory,
    File,
}

/// A representation of a filesystem object. For now, it only contains important metadata.
/// This is an initial implementation and will change as the project progresses.
#[derive(Clone, Debug, Serialize, Deserialize)]
pub struct FsObject {
    pub kind: FsKind,
    pub date_created: SystemTime,
    pub size: u64,
    pub current_path: PathBuf,
}

/// Initial implementation of the `FsObject` methods.
impl FsObject {
    /// This function creates a new file and a new `FsObject`. It's a wrapper from filesystem operations for creating files and directories from the `std::fs` module.
    pub fn create_file(target_path: &Path, _overwrite: bool) -> io::Result<Self> {
        // TODO: add feature to overwrite
        match fs::File::create_new(target_path) {
            Ok(result) => {
                let metadata = result.metadata()?;
                let fs_object = FsObject {
                    kind: FsKind::File,
                    size: metadata.size(),
                    date_created: metadata.created()?,
                    current_path: target_path.to_path_buf(),
                };
                Ok(fs_object)
            }
            Err(err) => Err(err),
        }
    }

    /// This function creates a new directory and a new `FsObject`. It's a wrapper from filesystem operations for creating files and directories from the `std::fs` module.
    pub fn create_dir(target_path: &Path) -> io::Result<Self> {
        match fs::create_dir_all(target_path) {
            Ok(_) => {
                let metadata = target_path.metadata()?;
                let fs_object = FsObject {
                    kind: FsKind::Directory,
                    size: metadata.size(),
                    date_created: metadata.created()?,
                    current_path: target_path.to_path_buf(),
                };
                Ok(fs_object)
            }
            Err(err) => Err(err),
        }
    }

    /// Creates a new FsObject from a path that must exist. If `None` was provided, it assumes the
    /// FsObject is at the current working directory.
    pub fn new(path: Option<PathBuf>) -> io::Result<FsObject> {
        let kind: FsKind;
        let size: u64;
        if let Some(p) = path {
            kind = if p.is_dir() {
                FsKind::Directory
            } else if p.is_file() {
                FsKind::File
            } else if !p.exists() {
                return Err(io::Error::new(
                    io::ErrorKind::NotFound,
                    "Source path to file does not exist!",
                ));
            } else {
                return Err(io::Error::new(
                    io::ErrorKind::Unsupported,
                    "Filesystem kind is not supported yet",
                ));
            };
            let metadata = p.metadata()?;
            let date_created = metadata.created()?;
            size = metadata.size();
            Ok(FsObject {
                kind,
                date_created,
                size,
                current_path: p,
            })
        } else {
            // Assume that it's the current working directory
            let current_directory = std::env::current_dir()?;
            let metadata = current_directory.metadata()?;
            let date_created = metadata.created()?;
            size = metadata.size();
            Ok(FsObject {
                kind: FsKind::Directory,
                date_created,
                size,
                current_path: current_directory,
            })
        }
    }

    pub fn move_to(&mut self, target_path: &Path) -> io::Result<()> {
        // TODO: Add utility functions here for moving directories.
        match self.kind {
            // WARNING: Destructive operation. On most file explorers, moving a directory is possible if the source directory
            // is empty and the target directory is also empty or does not exist. Our function's behaviour is different since
            // 1. It will delete the target directory first if it exists.
            // 2. Copies the source directory alongside its contents to the target directory.
            // 3. Deletes the source directory after.
            // This behaviour is not ideal but it will do for now.
            FsKind::Directory => {
                // We first delete the target directory, `target_path`, regardless if it is empty or not.
                if target_path.is_dir() && target_path.exists() {
                    fs::remove_dir_all(target_path)?;
                }
                copy_dir_all(&self.current_path, target_path)?;
                fs::remove_dir_all(&self.current_path)?;
                self.current_path = target_path.to_path_buf();
            }
            FsKind::File => {
                // The `fs::rename` function is oddly named but it's actually just renaming the path instead of the filename.
                // WARNING: Destructive operation. This will replace the file at the `target_path` if it exists.
                fs::rename(&self.current_path, target_path)?;
                self.current_path = target_path.to_path_buf();
            }
        }
        Ok(())
    }

    pub fn copy_to(&self, target_path: &Path) -> io::Result<()> {
        match self.kind {
            FsKind::Directory => copy_dir_all(&self.current_path, target_path),
            FsKind::File => match fs::copy(&self.current_path, target_path) {
                // NOTE: We might want to print debug statements in the future.
                Ok(_) => Ok(()),
                Err(err) => Err(err),
            },
        }
    }

    /// This function transfers ownership within the method. Since ownership is transferred,
    /// the value is moved out of scope and never returned.
    pub fn delete(self) -> io::Result<()> {
        if self.current_path.is_file() {
            fs::remove_file(&self.current_path)?;
        } else if self.current_path.is_dir() {
            // NOTE: Should we remove all contents within a directory or remove them
            // only if it's empty?
            fs::remove_dir_all(&self.current_path)?;
        } else {
            return Err(io::Error::other("Operation is not supported"));
        };
        Ok(())
    }

    /// This is just a wrapper around `.exists()` method from `std::path::Path`.
    pub fn exists(&self) -> bool {
        self.current_path.exists()
    }

    /// This is a wrapper around the `super::fs_wrappers::list` function.
    pub fn list(&self) -> io::Result<Vec<PathBuf>> {
        super::fs_wrappers::list(&self.current_path)
    }

    /// This is a wrapper around the `super::fs_wrappers::list_with_details` function.
    pub fn list_with_details(&self) -> io::Result<Vec<FsObject>> {
        super::fs_wrappers::list_with_details(&self.current_path)
    }
}

/// This function will list directories and files at the top-level of the target path `p`.
pub fn list(p: &Path) -> io::Result<Vec<PathBuf>> {
    if p.is_file() {
        // If the path is a file, only list the file.
        Ok(vec![p.to_path_buf()])
    } else if p.is_dir() {
        let read_dir = p.read_dir()?;
        let result: Vec<PathBuf> = read_dir
            .into_iter()
            .flatten()
            .map(|dir_entry| dir_entry.path())
            .filter(|dir| dir.exists())
            .collect();
        return Ok(result);
    } else {
        return Err(io::Error::other("Cannot list specified path!"));
    }
}

/// This function will list directories and files but in a more detailed form.
pub fn list_with_details(p: &Path) -> io::Result<Vec<FsObject>> {
    let list: Vec<PathBuf> = super::fs_wrappers::list(p)?;
    Ok(list
        .iter()
        .flat_map(|path| FsObject::new(Some(path.to_path_buf())))
        .collect())
}

    You can observe here that the #[derive()] attribute is on top of our custom structs and enums, FsObject and FsKind. The attribute allows you to derive from trait implementations without the need to modify code and without the need to write more boilerplate code.

    One of the important trait implementations to derive into the custom types are Serialize and Deserialize.

    Serialization is the process of converting a Rust data structure into a format that can be stored or transmitted (e.g., turning a struct into a JSON string).

    Deserialization is the reverse process—converting a format like a JSON string back into a Rust data structure.

    use axum::{Json, http::StatusCode, response::IntoResponse};
use serde::{Deserialize, Serialize};
use std::path::PathBuf;

use crate::fs_wrappers::FsObject;

#[derive(Serialize, Deserialize, Debug)]
pub enum FileOpts {
    Move,
    Copy,
    Delete,
    CreateFile,
    CreateDirectory,
}

#[derive(Deserialize)]
pub struct UserCommand {
    pub file_opts: FileOpts,
    pub source_path: PathBuf,
    #[serde(default)]
    pub target_path: Option<PathBuf>,
}

#[derive(Serialize, Deserialize, Debug)]
pub enum FileOptStatus {
    Failed(FileOpts),
    Success(FileOpts),
    NoUserInput,
}

pub async fn list() -> (StatusCode, Json<Vec<FsObject>>) {
    let current_dir = std::env::current_dir().unwrap();
    let fs_object = FsObject::new(Some(current_dir)).unwrap();
    let fs_objects = fs_object.list_with_details().unwrap();
    (StatusCode::OK, Json(fs_objects))
}

pub async fn list_fs_objects(Json(payload): Json<Option<PathBuf>>) -> impl IntoResponse {
    let current_dir = std::env::current_dir().unwrap();
	if let Some(source_path) = payload {
        let fs_object = FsObject::new(Some(source_path)).unwrap();
        let list_fs_objects = fs_object.list_with_details().unwrap();
        (StatusCode::OK, Json(list_fs_objects))
    } else {
        let fs_object = FsObject::new(Some(current_dir)).unwrap();
        let list_fs_objects = fs_object.list_with_details().unwrap();
        (StatusCode::OK, Json(list_fs_objects))
    }
}

pub async fn perform_file_opt(Json(payload): Json<UserCommand>) -> impl IntoResponse {
    let user_command: UserCommand = payload;

    // Rigorously check if inputs are there.
    match &user_command.file_opts {
        FileOpts::Move | FileOpts::Copy => {
            if user_command.target_path.is_none() {
                return (
                    StatusCode::METHOD_NOT_ALLOWED,
                    Json(FileOptStatus::NoUserInput),
                );
            }
            if let Some(target_path) = &user_command.target_path {
                if target_path.to_string_lossy().is_empty() {
                    return (
                        StatusCode::METHOD_NOT_ALLOWED,
                        Json(FileOptStatus::NoUserInput),
                    );
                }
            }
        }
        _ => {
            if user_command.source_path.to_string_lossy().is_empty() {
                return (
                    StatusCode::METHOD_NOT_ALLOWED,
                    Json(FileOptStatus::NoUserInput),
                );
            }
        }
    }

    // We already covered the error-handling above anyway, so unwrapping here
    // is okay.
    let target_path = user_command.target_path.unwrap_or_default();

    match user_command.file_opts {
        FileOpts::Move => {
            let mut fs_object =
                FsObject::new(Some(user_command.source_path.to_path_buf())).unwrap();
            if fs_object.move_to(&target_path).is_ok() {
                (StatusCode::OK, Json(FileOptStatus::Success(FileOpts::Move)))
            } else {
                (
                    StatusCode::INTERNAL_SERVER_ERROR,
                    Json(FileOptStatus::Success(FileOpts::Move)),
                )
            }
        }
        FileOpts::Copy => {
            let fs_object = FsObject::new(Some(user_command.source_path.to_path_buf())).unwrap();
            if fs_object.copy_to(&target_path).is_ok() {
                (StatusCode::OK, Json(FileOptStatus::Success(FileOpts::Copy)))
            } else {
                (
                    StatusCode::INTERNAL_SERVER_ERROR,
                    Json(FileOptStatus::Failed(FileOpts::Copy)),
                )
            }
        }
        FileOpts::Delete => {
            let fs_object = FsObject::new(Some(user_command.source_path.to_path_buf())).unwrap();
            if fs_object.delete().is_ok() {
                (
                    StatusCode::OK,
                    Json(FileOptStatus::Success(FileOpts::Delete)),
                )
            } else {
                (
                    StatusCode::INTERNAL_SERVER_ERROR,
                    Json(FileOptStatus::Failed(FileOpts::Delete)),
                )
            }
        }
        FileOpts::CreateFile => {
            if FsObject::create_file(&user_command.source_path, false).is_ok() {
                (
                    StatusCode::OK,
                    Json(FileOptStatus::Success(FileOpts::CreateFile)),
                )
            } else {
                (
                    StatusCode::INTERNAL_SERVER_ERROR,
                    Json(FileOptStatus::Failed(FileOpts::CreateFile)),
                )
            }
        }
        FileOpts::CreateDirectory => {
            if FsObject::create_dir(&user_command.source_path).is_ok() {
                (
                    StatusCode::OK,
                    Json(FileOptStatus::Success(FileOpts::CreateDirectory)),
                )
            } else {
                (
                    StatusCode::INTERNAL_SERVER_ERROR,
                    Json(FileOptStatus::Failed(FileOpts::CreateDirectory)),
                )
            }
        }
    }
}

    Head to the root directory of the project. Head to the serve project. Run the following commands inside the serve directory.

    cargo add tokio -F full

    This should add tokio and axum crates, as well as the libfile-explorer crate.

    Open and edit the file at src/main.rs. Replace its contents with the following code.

    use axum::routing::delete;
use axum::Router;
use axum::routing::post;
use axum::routing::get;
use libfile_explorer::json_api::copy_opt;
use libfile_explorer::json_api::create_dir;
use libfile_explorer::json_api::create_file;
use libfile_explorer::json_api::delete_opt;
use libfile_explorer::json_api::list_fs_objects;
use libfile_explorer::json_api::move_opt;
use libfile_explorer::json_api::perform_file_opt;

#[tokio::main]
async fn main() {
    let app = app();
    let listener = tokio::net::TcpListener::bind("0.0.0.0:3000").await.unwrap();

fn app() -> Router {

        .route("/api/operate/delete", delete(delete_opt))
        .route("/api/create/file", post(create_file))
        .route("/api/create/directory", post(create_dir))
}

  • Step 5: Run the server code and using curl

    Ensure that you have curl installed, otherwise, head to https://curl.se/download.html for installation instructions.

    cargo run --release -p serve

    Let's try creating a file and a directory.

    Run the following command. This will create a new file called "new_file" at the root directory of the project. Running this command again should give you a status message that it failed.

    curl -XPOST -H "Content-type: application/json" -d '"new_file"' --url "http://localhost:3000/api/create/file"

    Run the following command. This will create a new directory called "new_directory" at the root directory of the project. Running this command again should still give you a success status.

    curl -XPOST -H "Content-type: application/json" -d '"new_directory"' --url "http://localhost:3000/api/create/directory"

    Let's try copying a file and a directory.

    Run the following command. This will copy the directory called "new_directory" as "copied_directory".

    curl -XPOST -H "Content-type: application/json" -d '["new_directory", "copied_directory"]' --url "http://localhost:3000/api/operate/copy"

    Run the following command. This will copy the file called "new_file" to another file called "copied_file" inside "copied_directory".

    curl -XPOST -H "Content-type: application/json" -d '["new_file", "copied_directory/copied_file"]' --url "http://localhost:3000/api/operate/copy"

    Let's try moving a directory.

    Run the following command. This will replace the existing "copied_directory" including its contents. Running the command again will cause a bug. The bug will delete the "copied_directory". This bug also happens to files as well. Take note of this bug as it will be part of a challenge.

    curl -XPOST -H "Content-type: application/json" -d '["new_directory", "copied_directory"]' --url "http://localhost:3000/api/operate/move"

    To delete a file or directory, run the following command. Running the command again will give you a curl command error. This is a bug since an error should still give you a reply from the server. Take note of this bug as it will be part of a challenge.

    curl -i -XDELETE -H "Content-type: application/json" -d '"new_file"' --url "http://localhost:3000/api/operate/delete"

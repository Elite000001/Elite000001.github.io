Are We Web Yet?

7 steps

In this overview, we will be inviting you to explore some of the web technologies that are popular in the Rust web ecosystem.

Rusts web ecosystem has been growing and maturing at a fast-pace! Be sure to check out during your free time!

Learning Outcomes

By the end of this quest, you will be able to:

  • Learn popular third-party web crates
  • Know why Rust is whether Web-ready or not
  • Ideate for what crates to use for your future Rust projects
  • Get to know upcoming and new technologies in Rusts web ecosystem

Tutorial Steps

Total steps: 7

  • Step 1: Is Rust Web-Ready Yet?

    Rust has been hyper-focused (pun intended) on having a mature ecosystem in Web technologies.

    Within a decade ago before this tutorial was posted, Go has been the leading force when it comes to web technologies and tooling such as Hugo for building content and static websites such as blogs, Docker for development containers, Kubernetes for launching thousands of web services, and Caddy for hosting HTTP servers.

    So how is Rust doing?

    Rust has been accelerating development but more focused on backend web services and web authentication such as jsonwebtoken crate, and axum!

    Alright? So now what? Lets meet some of these crates in the next steps and let us know what you think at the StackUp Discord channel.

  • Step 2: Meet Hyper

    Hey, is this the pun? Meet https://hyper.rs, the Rust crate used as the core for popular web frameworks in Rust such as axum and warp!

    Its a relatively low-level library so if you want to build something from scratch such as a server framework, this is a good library to start. But if you want to build something without the need of adding low-level boilerplate, axum and warp are good options.

  • Step 3: Meet Reqwest

    Created by the same author of https://hyper.rs, reqwest is a powerful Rust HTTP Client, used to build API libraries or even for simple HTTP requests.

    If you have gone to the Learn Pathway for Rust, you might have visited Building an API Client Library with Rust I. The tutorial uses reqwest to build the API for https://endoflife.date. If you havent tried the tutorial, be sure to visit and so you can use reqwest in action!

  • Step 4: Meet Axum

    Axum is a web framework designed for building robust and scalable web applications. Developed by the team at Tokio, Axum is a relatively modern framework that aims to provide a simple and intuitive API for creating web services.

    Wait whats tokio? Tokio is an asynchronous Rust runtime framework built for networking services. It allows multi-thread safety and allows parallelisation so that your services are always up and running. Of course, you have to build that logic so you can allow that to happen

    For how axum is used would probably look like the code below

    use axum::{
	extract::Path,
	routing::{get, Router},
	response::Html,
	Json,
	http::StatusCode,
};
use serde_json::json;
use std::net::SocketAddr;

#[tokio::main]
async fn main() {
	// Build our application with a route
	let app = Router::new()
    	.route("/", get(root))
    	.route("/greet/:name", get(greet))
    	.route("/json", get(json_response));

	// Define the address and port for our server
	let addr = SocketAddr::from(([127, 0, 0, 1], 3000));

	// Start the server
	println!("Listening on http://{}", addr);
	axum::Server::bind(&addr)
    	.serve(app.into_make_service())
    	.await
    	.unwrap();
}

// A handler for the root route
async fn root() -> Html<&'static str> {
	Html("<h1>Hello, world!</h1>")
}

// A handler that greets the user by name
async fn greet(Path(name): Path<String>) -> String {
	format!("Hello, {}!", name)
}

// A handler that returns a JSON response
async fn json_response() -> Json<serde_json::Value> {
	Json(json!({
    	"message": "Hello, JSON!",
    	"status": "success"
	}))
}

  • Step 5: Are There Any Fullstack Frameworks in Rust?

    Oh there are! Rust has a fast growing web ecosystem, it would be no surprise that it already has frameworks that are alternatives to

    - Electron

    - React

    Electron Alternative - Tauri

    Do you already have an existing Web Application project? Do you want it to look platform-native? Try out Tauri!

    Tauri can be even used for Web projects that only use HTML, CSS and JS without the need of a framework! Try it out now!

    Visit https://github.com/tauri-apps/awesome-tauri for more awesome Tauri stuff!

    React-like Framework Alternative - Dioxus

    But I want React but in Rust!? Hey, Rusts open source community has Dioxus!

    This React-like framework in Rust uses an advanced web technology called web assembly thats still in the process of adoption so that you can build off native cross platform applications in just one build process! It provides a full framework and can be integrated with other frameworks such as axum and tower!

    Visit their website and hop into their documentation to build your first Dioxus app and join their Discord!

  • Step 6: Web Assembly? Whats That?

    WebAssembly (often abbreviated as wasm) is a binary instruction format designed to enable high-performance execution of code on web browsers and other platforms. It provides a portable compilation target for programming languages, allowing code written in languages like C, C++, and Rust to run efficiently on the web.

    WebAssembly code executes at near-native speed by leveraging modern hardware capabilities, and it is designed to work alongside JavaScript, making it possible to enhance web applications with performance-critical components while maintaining the flexibility and ease of development associated with JavaScript.

    Its still considered experimental but the technology has been maturing very fast!

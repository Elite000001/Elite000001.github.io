Practical Applications of Vector Search

6 steps

In this quest, youll explore the practical applications of vector search through text-to-text matching. Instead of working with images, this quest focuses on how we can apply vector search techniques to textual data. Using pre-trained models like SentenceTransformers, youll generate vector embeddings for a collection of text entries, store them in ChromaDB, and perform searches based on vector similarity.

Text embeddings capture the semantic meaning of sentences, allowing us to perform intelligent searches based on context, not just exact keyword matches.

Learning Outcomes

By the end of this quest, you will be able to:

  • Understand the key concepts of vector search and how it applies to text-to-text matching.
  • Implement a basic text-to-text search system using pre-trained models to generate vector embeddings.
  • Use ChromaDB to store and query embeddings efficiently, facilitating fast and accurate similarity searches.
  • Evaluate the accuracy and relevance of search results based on vector similarity.

Tutorial Steps

Total steps: 6

  • Step 1: Environment Setup

    In this quest, you will implement text-to-text vector search by coding a basic pipeline in Python using pre-trained models to generate embeddings and ChromaDB to store and search these embeddings.

    Open your terminal and navigate to the root of your project folder created from Quest 1. Activate the .venv if it isnt already by executing the following command:

    On macOS/ Linux:

    source .venv/bin/activate
    Plain text

    On Windows:

    .venv\Scripts\activate
    Plain text

    With the virtual environment activated, install additional packages that will be needed for this script.

    To install the transformers library, which will be used for generating embeddings with pre-trained models, run the following command:

    pip install git+https://github.com/huggingface/transformers
    Plain text

    Next, install the sentence-transformers library, which provides additional functionality for working with text embeddings:

    pip install sentence-transformers
    Plain text

    With these libraries installed, your environment is now ready to proceed with setting up ChromaDB.

  • Step 2: Setting Up ChromaDB

    Now that your environment is set up, you can begin coding the text-to-text vector search. In this step, you will initialize ChromaDB and create a collection to store the text embeddings. This is a foundational setup that will allow you to efficiently store and retrieve vectors for similarity searches.

    In your IDE, make sure you are at the root of your project directory. Create a new Python script and name it text-to-text.py. Open the file and start by importing the necessary libraries.

    Write the following code block:

    from sentence_transformers import SentenceTransformer
import chromadb
    Plain text

    Initialize ChromaDB and create a new collection where your text embeddings will be stored. The collection will act like a database table, organizing your embeddings and allowing you to perform efficient queries.

    On the next lines, add the following code:

    client = chromadb.Client()
collection = client.create_collection("text_collection")
    Plain text
    • ChromaDB Client: The client = chromadb.Client() line initializes the ChromaDB client, which is responsible for all interactions with the database. This client will help you store, retrieve, and query vector data.
    • Collection: A collection in ChromaDB is like a table in a traditional database. It stores embeddings (vectors) and their associated metadata. By creating a collection, you're defining a space where your text vectors will be inserted and searched. Here, text_collection is the name of your collection, which will hold the vectors for this project.

    This step sets the foundation for how your data will be stored and queried efficiently in ChromaDB. Without this setup, you wouldn't be able to store or retrieve embeddings in the upcoming steps.

  • Step 3: Loading the Pre-Trained Embedding Model and Ingesting Text Data

    Now that you've set up your ChromaDB client and collection, its time to load a pre-trained embedding model and ingest text data into your collection. This step involves generating vector representations (embeddings) for multiple text entries and adding them to the ChromaDB collection for future querying.

    To generate embeddings for text data, you will use the pre-trained model all-MiniLM-L6-v2, which is a compact model that provides high-quality sentence embeddings.

    On the next lines, write the following code block to load the model:

    model = SentenceTransformer('all-MiniLM-L6-v2')
    Python

    The SentenceTransformer model transforms text into embeddings that capture the semantic meaning of sentences, which makes it suitable for vector search tasks.

    Next, youll add multiple text entries that represent different types of content, ranging from iconic landmarks to scientific discoveries and everyday scenes.

    On the next lines, write the following code block to define your text data:

    texts = [
    "The Eiffel Tower in Paris is an iconic landmark.",
    "A delicious homemade pizza fresh from the oven.",
    "A scientist studying the effects of climate change on polar bears.",
    "A group of friends hiking through the mountains during autumn.",
    "The discovery of water on Mars could mean the possibility of life.",
    "The rise of artificial intelligence is reshaping industries.",
    "A peaceful garden filled with blooming flowers and chirping birds.",
    "A musician playing the violin in a crowded subway station.",
    "A historical novel set during World War II.",
    "An astronaut floating weightlessly in space.",
     "The space shuttle launched into orbit, marking a new era in space exploration.",
    "An advanced AI system is helping doctors diagnose rare diseases.",
    "A cozy cafe in the city center where people gather to relax and chat."
]
    Python

    This array of text entries is designed to provide diverse content that you will later query using vector search. Each entry will be converted into a vector and added to your collection in ChromaDB.

    Youll now generate embeddings for each text entry and insert those embeddings into your ChromaDB collection. These embeddings are vector representations of the text that capture the key features and relationships between them.

    On the next lines, write the following code block:

    embeddings = model.encode(texts)

collection.add(
    embeddings=embeddings,
    metadatas=[{"text": text} for text in texts],
    ids=[str(i) for i in range(len(texts))]
)
    Python

    Code Breakdown:

    • model.encode(texts) generates a vector (embedding) for each text entry in the texts list.
    • The collection.add() function inserts the embeddings, along with metadata (which in this case includes the original text), into the collection. The ids parameter assigns unique identifiers to each entry.

    By completing this step, you've successfully ingested text data into ChromaDB, and it is now ready for vector search queries. You'll be able to search for similar text entries based on embeddings, which will be demonstrated in the upcoming steps.

  • Step 4: Making Queries and Displaying the Results

    Now that you have ingested text data into your ChromaDB collection, you can test the vector search functionality by making queries and displaying the results.

    You will create a few queries that represent different concepts. These queries will be converted into embeddings, and the system will search for the most similar entries in the database based on their vector proximity.

    Write the following code block on the next lines to define the queries:

    queries = [
    "A musician performing in public",
    "A peaceful place with flowers and birds",
    "Medical professionals using machine learning to find rare illnesses", 
    "Astronaughts traveling to outside of earth",
    "Relaxing at a shop with friends",
]
    Python

    These queries are diverse and are designed to test the model's ability to retrieve similar text entries based on the meaning of the queries.

    Next, you will iterate through the defined queries and perform vector searches to find the top 3 most similar entries in your collection.

    Write the following code block on the next lines:

    for query in queries:
    print(f"\nQuery: {query}")
    
    # Generate an embedding for the query text
    query_embedding = model.encode(query)

    # Perform a vector search in the collection
    results = collection.query(
        query_embeddings=[query_embedding],
        n_results=3  # Retrieve top 3 similar entries
    )
    Python

    Heres whats happening:

    • The model.encode(query) generates a vector for each query.
    • The collection.query() function performs a vector search, retrieving the top 3 entries from the collection that are most similar to the query.

    Finally, you will display the results from the search by retrieving the text metadata associated with each result.

    Add the following code block at the end of the previous for loop:

    for i, result in enumerate(results['metadatas'][0]):
         print(f"Result {i + 1}: {result['text']}")
    Python

  • Step 5: Analysis of the Vector Search Results

    In this step, we shift focus from how vector search works to evaluating the quality of the results it produces. The goal is to assess how well the retrieved results match the intended meaning of the query. Well look at both strong matches, where the system accurately captures the query's context, and weaker matches, where the system stretches the meaning or draws less relevant connections. 

    Through this analysis, we can better understand how vector embeddings work in practice and identify ways to improve search accuracy using more sophisticated models or techniques.

    Query 1: A peaceful place with flowers and birds

    Result 1.A: "A peaceful garden filled with blooming flowers and chirping birds."

    This is a perfect match. The query asks for a peaceful setting with flowers and birds, and the result describes a garden with those exact elements. The model recognized the overall meaning and provided a nearly identical context.

    Result 1.B: "A cozy cafe in the city center where people gather to relax and chat."

    This result is a good match because it describes a cozy and relaxing setting, which is thematically similar to the peaceful environment in the query. The model understood the concept of "peaceful" as related to "cozy" and "relaxing," even though there are no flowers or birds in the text.

    Result 1.C: "A historical novel set during World War II."

    This result is an outlier. The model may have picked this up because "peaceful" could be interpreted as the opposite of "war," creating a very loose thematic connection. However, it's not an ideal match.

    Query 2: Medical professionals using machine learning to find rare illnesses

    Result 2.A: "An advanced AI system is helping doctors diagnose rare diseases."

    This is an excellent match. The query talks about using machine learning (a form of AI) in healthcare, and the result describes an AI system diagnosing rare diseases, which is exactly what the query was asking for, even though the wording is different.

    Result 2.B: "The rise of artificial intelligence is reshaping industries."

    This is a good match because it talks about AI, which relates to machine learning. While this result focuses on AI in general industries, the model connected the broader theme of AI to the query, which focused on healthcare.

    Result 2.C: "The discovery of water on Mars could mean the possibility of life."

    This result is not directly related, but it could have been included because the model identified "discovery" and "possibility of life" as themes related to innovation and breakthrough, which loosely connects with the idea of finding solutions using machine learning.

    Query 3: Astronauts traveling to outside of earth

    Result 3.A: "The space shuttle launched into orbit, marking a new era in space exploration."

    This is a perfect match. The query is asking about astronauts traveling beyond Earth, and this result describes a space shuttle launching into orbit, which is exactly what the query implies.

    Result 3.B: "The discovery of water on Mars could mean the possibility of life."

    This is a relevant match because it talks about Mars, which is outside Earth, and space exploration is a key theme here. The model connected the context of space exploration to the query, even though the specific mention of astronauts is missing.

    Result 3.C: "A group of friends hiking through the mountains during autumn."

    This is an unusual result, but the model might have linked the theme of traveling to the hiking activity, which involves exploration, though in a very different context. This result shows how the model sometimes stretches to find connections.

    Summary of Results:

    • Strong Matches: In many cases, the model successfully identifies the meaning and context of the query, even when the specific words differ. For example, "machine learning" was matched with "AI," and "a peaceful place" was matched with a garden and cafe.
    • Loose Connections: Occasionally, the model stretches meaning, pulling in results that seem more loosely related, like the hiking result for "astronauts traveling" or the pizza result for "relaxing with friends." These results show how the model seeks contextual links, sometimes broadening the scope of the search.

    To reduce loose connections in vector search, choosing a more sophisticated or domain-specific model can improve result precision. For instance, upgrading from all-MiniLM-L6-v2 to models like paraphrase-distilroberta-base-v1 or msmarco-distilbert-base-v4 can capture more nuanced meaning. Fine-tuning on a specific dataset tailored to your content, such as scientific or healthcare text, will further improve contextual accuracy. 

    Additionally, higher-dimensional embeddings, such as those from paraphrase-MiniLM-L12-v2, can provide deeper semantic understanding. Using metadata filtering to narrow results and reranking with cross-encoder models like cross-encoder/ms-marco-TinyBERT-L6 can further refine results, reducing irrelevant matches. These combined strategies can significantly enhance the accuracy of vector searches.

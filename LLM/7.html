Creating a Chatbot with Gemini API

7 steps

In this tutorial, you will learn how to build a dynamic chatbot powered by Google's Gemini Pro artificial intelligence model, leveraging the robust capabilities of Next.js and React. This chatbot will not only respond intelligently to user input but also feature rich text formatting and quick response functionalities to enhance user interaction.

You will set up a fully functional web application that communicates with the Gemini Pro model to process and output text, learn how to handle API requests securely, and apply JavaScript and CSS techniques to produce a responsive and visually appealing interface. By the end of this tutorial, you will have developed a comprehensive understanding of integrating AI into web applications and deploying them live for user interaction.

  • Step 1: What is a Chatbot

    A chatbot, also known as a conversation bot, is an artificial intelligence (AI) program designed to simulate conversation with human users through text or voice interactions. These AI-driven systems are programmed to understand and respond to user queries, providing information, assistance, or performing specific tasks. Chatbots can be found in various contexts, from customer service applications to smart speakers at home. They serve several purposes:

    1. Customer Support: Chatbots can handle common customer inquiries, providing instant responses and freeing up human agents for more complex issues. This improves efficiency and reduces wait times.
    2. Information Retrieval: Users can ask chatbots questions, and they retrieve relevant information from databases, websites, or knowledge bases. For example, chatbots on websites often assist with FAQs.
    3. Personal Assistants: Virtual chatbots act as personal assistants, helping users manage their calendars, set reminders, and perform other daily tasks.

    AI enhances chatbots by enabling more sophisticated interactions. Heres how:

    1. Generative AI: The next generation of chatbots uses generative AI, which goes beyond predefined answers. These chatbots generate new content based on language models, adapting to users conversational style and context.

    2. Self-Learning: AI chatbots learn from past interactions, improving their responses over time. They adapt to user preferences and continuously enhance their understanding.

    In summary, chatbots powered by AI offer efficient, personalized, and scalable solutions across various domains, making them valuable tools for businesses and users alike.

    import React, { useState } from 'react';
import styles from '../styles/Home.module.css'; // Ensure the path to your CSS file is correct

export default function Home() {
    const [messages, setMessages] = useState([]);
    const [input, setInput] = useState('');
    const [showQuickResponses, setShowQuickResponses] = useState(true);

    return (

                    onKeyDown={handleKeyDown}
                    className={styles.input}

            </div>
        </div>
    );
}

    Next, we will be writing some functions to handle user input and the sending of messages.

    Immediately after the states (line 8), insert the following code.

        const handleInputChange = (e) => {

    JavaScript

    First, we have the event handler that updates the input state as the user types into the input field. Then, we have the event handler for the input field, to allow sending messages by pressing the Enter key, which enhances the user experience. Finally, we have the quick response function. In our chatbot, these are three buttons with preset prompts that a user can click on, in the event that they do not know how to start a conversation with the chatbot. These buttons use the sendMessage function which we will code next.

    Next, copy and paste the following code.

    const sendMessage = async (message) => {
        const trimmedMessage = message.trim();
        if (!trimmedMessage) return;

        setShowQuickResponses(false);

        const newMessage = { id: Date.now(), text: trimmedMessage, sender: 'user' };
        setMessages(messages => [...messages, newMessage]);
        setInput('');

        const response = await fetch('/api/chat', {
            method: 'POST',
            headers: {
                'Content-Type': 'application/json',
            },
            body: JSON.stringify({ message: trimmedMessage }),
        });
        const { reply } = await response.json();
        const formattedReply = formatText(reply);

        setMessages(messages => [...messages, { id: Date.now() + 1, text: formattedReply, sender: 'bot' }]);
    };
    
 const formatText = (text) => {
        let formattedText = text.replace(/\*\*(.*?)\*\*/g, '<strong>$1</strong>');
        formattedText = formattedText.split('\n').map(line => {
            if (line.trim().startsWith('* ')) {
                return `<li>${line.trim().substring(2)}</li>`;
            }
            return line;
        }).join('\n');

        if (formattedText.includes('<li>')) {
            formattedText = `<ul>${formattedText}</ul>`;
        }

        return formattedText;
    };
    JavaScript

    When we send our message, we first want to trim it by removing any leading or trailing whitespace. This is a common practice to ensure that empty messages (consisting only of whitespace) are not processed. Once the message is sent, we want to hide our quick response buttons by setting the state to false.

    A newMessage object is created with the current timestamp as an identifier (id), the trimmed message text, and the sender labelled as 'user'. This message is then added to the messages state, which triggers a re-render of the UI to display this new message. The input field is cleared to prepare for the next user input.

    The trimmed message is sent to the server using the fetch API with a POST request. This request targets the endpoint to handle chat messages (/api/chat). We will code this in the next step. The message is included in the request body as a JSON string. Proper headers (Content-Type: application/json) are set to ensure the server interprets the request body correctly.

    The function waits for the response from the server and then extracts the JSON body from it. The expected response includes a field named reply, which contains the AI-generated response. This response is then passed to the function formatText, which is the second function in the above code snippet. Finally, a new message object is created for the AI response, similar to the user message but labelled as coming from 'bot'. It includes the formatted reply. This message is added to the messages state, updating the UI to display the new AI-generated message.

    Next, the formatText function is meant to provide some rich text formatting to the response from our chatbot. With LLMs, their responses often include bolded text (denoted by **) and bullet points (denoted by *). We use regex to check for words surrounded by two asterisks and ensure they are bolded. Similarly, we look for lines that start with a single asterisk and ensure that it becomes a list element.

    Next, copy and paste the following code.

        const renderMessage = (msg) => {
        if (msg.sender === 'bot') {
            return <p key={msg.id} className={styles.botMessage} dangerouslySetInnerHTML={{ __html: msg.text }}></p>;
        } else {
            return <p key={msg.id} className={styles.userMessage}>{msg.text}</p>;
        }
    };
    JavaScript

    The renderMessage function is a React component helper function designed to render chat messages based on who the sender is.

    The function checks the sender property of the message to determine how the message should be rendered. If the sender is 'bot', it renders the message using a <p> element with a class of botMessage, which applies specific styling for messages from the bot. This <p> element uses the dangerouslySetInnerHTML property to insert the message text as HTML into the DOM. This approach renders potentially complex HTML content that includes formatted text and other HTML elements. However, it's called "dangerous" because it opens up potential risks of cross-site scripting (XSS) attacks if the content isn't properly sanitized. On the other hand, if the sender is not 'bot' (typically 'user'), it renders the message text directly within a <p> element with a class of userMessage. This direct insertion is safe from XSS because React automatically escapes any dangerous characters in the text, preventing HTML from being rendered unless explicitly done so.

    With that, we are done with our code for index.js.

    In your terminal, ensure that you are in the my-chatbot directory. If you are not, you can change directory by using

    cd my-chatbot

    Then, you can install the SDK package by running the following command.

    npm install @google/generative-ai

    Next, in the pages/api folder, create a new file named chat.js. This is the file where we will set up an API route in Next.js that communicates with Google Gemini LLM. Copy and paste the following code.  

    import { GoogleGenerativeAI } from "@google/generative-ai";

const genAI = new GoogleGenerativeAI(process.env.API_KEY);

export default async function handler(req, res) {
  if (req.method === "POST") {
    const userMessage = req.body.message;

    try {
      const model = genAI.getGenerativeModel({ model: "gemini-1.0-pro" });
      const result = await model.generateContent(userMessage);
      const response = await result.response;
      const text = await response.text();

      res.status(200).json({ reply: text });
    } catch (error) {
      console.error("Error processing the AI response:", error);
      res.status(500).json({ error: "Failed to process the AI response" });
    }
  } else {
    res.setHeader("Allow", ["POST"]);
    res.status(405).end(`Method ${req.method} Not Allowed`);
  }
}
    JavaScript

    First, we import and initialize the generative model. With the API key, we can interact with Googles AI services. Then, we create a request method check to check if the incoming request is a POST request, ensuring that the handler only processes requests that submit data (in this case, messages to the chatbot).

    Then, we interact with our AI model. We state the model that we are using. In our case, we use the Gemini 1.0 Pro model which takes text as an input and outputs text. It currently has a rate limit of 60 requests per minute, which should be more than sufficient. There are other models that you can use and you can view the documentation here.

    We then wait for a response. Only the text is extracted from the response. If the previous steps complete without errors, the server sends back a 200 OK HTTP status with the AI-generated text in the response body.

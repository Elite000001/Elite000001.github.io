Build Your Own Docker Image

3 steps

From the previous tutorial, you've learned how to use the Docker CLI commands such as docker pull, docker run and docker ps. In this tutorial, you will be building your own docker image with Dockerfile and running your created image as a Docker container.
Join us on Discord

Helpful prior knowledge

  • There are no pre-requisites

Learning Outcomes

By the end of this tutorial, you will be able to:

  • Create a Dockerfile
  • Build a Docker Image from your Dockerfile and tag the image
  • Run Docker Container from custom image

Tutorial Steps

Total steps: 3

  • Step 1: Create a Dockerfile

    The Dockerfile is a set of instructions for creating a Docker image. You've been using default Docker images from Docker Hub so far. With a Dockerfile, you can define your desired instructions and create your own custom docker images tailored to your needs.

    Let’s start by creating a Dockerfile. The IDE used in this tutorial will be VSCode. If you do not have a preferred IDE, we recommend Visual Studio Code (VSCode).

    First, create a directory under your desktop following the steps below.

    cd Desktop
mkdir intro_to_docker
cd intro_to_docker

    Next, navigate to your VSCode and open the intro_to_docker folder newly created.

    Then, let’s create a simple index.html file that you will host through Nginx using a Docker container.

    <html>
<h1>Hello Stackies</h1>
<p>This is your custom NGINX Docker container</p>
</html>

    IMPORTANT: Replace “Stackies” in the code above with your StackUp username.

    Next, create a new file titled Dockerfile with the following code.

    FROM nginx:latest
COPY ./index.html /usr/share/nginx/html/index.html
    Docker

    Here's how your Dockerfile should look like!

    In order to build a custom image, a base image is required. On line 2, this is done using the FROM command pulling the latest Nginx Docker image from Docker Hub. The following instruction will define the custom Docker image on top of the base image.

    Next, using the COPY command, the index.html created will overwrite the default index.html provided by the Nginx image.

    You can read more about the different Dockerfile instructions in the link below:

    🔗 Dockerfile Documentation

    By using Dockerfiles, DevOps teams can create stable environments for development, production and staging. In doing so, it provides a high level of control over all changes, ensuring that if a feature is working in one environment, it will work in other environments as well. It also allows for image versioning when there is a new feature ready to be rolled out or rollbacks if necessary.

    With your Dockerfile ready, you are now ready to build out your custom Nginx Docker image!

  • Step 2: Build Your Custom Docker Image

    In Step 1, you have created an index.html file and a Dockerfile containing instructions to build out your custom Nginx Docker image.

    Next, run the command below to build your custom Docker image on the VSCode terminal.

    docker build -t <image_name> .
    Docker

    IMPORTANT: Remember to define your image name when running the docker build command

    Docker will proceed to build your custom docker image based on your created Dockerfile. The . option specifies that the path is at the current directory and the -t option allows you to define your custom docker image name for easier identification.

    You can run the docker image ls command to verify that your custom docker image has been built successfully!

  • Step 3: Run Your Custom Docker Container

    Now that you have your custom Nginx Docker image, it’s time to run your custom Nginx Docker container.

    On the terminal, run the command below:

    docker run -d -p 8080:80 --name <container-name> <image-name>
    Docker

    IMPORTANT: Remember to use the right image name when running the docker run command and define your container name.

    After running the docker run, proceed to run docker ps where it should show the right image tag.

    IMPORTANT: If you face the error that port is already allocated, you can stop the running nginx container from tutorial 2 using the Docker CLI or from Docker Desktop as the same port 8080 is used for both tutorials.

    Next, open your browser and navigate to https://localhost:8080 to make sure that the revised html page is rendered correctly.

    Congratulations Stackies, you have successfully now built your own custom Nginx Docker image with Dockerfile and run your custom Docker container.

    You can now navigate to Docker Desktop to stop and delete the Docker containers and images should you wish to.

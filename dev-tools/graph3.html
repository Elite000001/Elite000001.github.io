Fundamentals of GraphQL

5 steps

In the previous tutorial, you have completed a refresher on what REST API is and made a simple GET request using Postman. You also read up on GraphQL and discovered the benefits of GraphQL over REST APIs. Now it's time to dive deeper into the fundamental concepts of GraphQL such as queries, mutations, type definitions and resolver functions. You will also learn to build a GraphQL API with Node.js and Apollo.
Join us on Discord

Helpful prior knowledge

  • There are no pre-requisites

Learning Outcomes

By the end of this tutorial, you will be able to:

  • Describe what GraphQL Schemas are
  • Explain how GraphQL Queries and Mutations work
  • Explain how Resolver Functions work
  • Build a GraphQL API using Node.js and Apollo

Tutorial Steps

Total steps: 5

  • Step 1: Schema and Object Types in GraphQL

    Before we begin building GraphQL API, we must first understand what schemas and types are. A schema is used to describe the shape of data and enforce data integrity through field definition where you must first define your schema using a user-friendly schema definition language (SDL). The SDL is very similar to how schemas are created in databases.

    Object types are the most fundamental components of a GraphQL schema and contain information about the object and its fields. A schema, on the other hand, is a collection of various object types and their relationships.

    Below is a sample schema which we will reference to explain the concepts of types and schemas:

    type Student {
 studentId: Int!
 modulesEnrolled: [Module!]!
}

type Module {
 moduleId: Int!
 moduleTitle: String!
}
    GraphQL

    With reference to the above schema, Student and Module are object types, which means they are objects with fields. The studentId and modulesEnrolled fields are found in the Student type, while the moduleId and moduleTitle fields are found in the Module type.

    You will also notice the [Module!]! field type definition for the modulesEnrolled field and this represents an array of Module objects. The "!" notation indicates that the field is non-nullable, which means that the GraphQL service will always return a value when this field is queried.

  • Step 2: Queries, Mutation and Resolver Functions in GraphQL

    Next, we will be looking at operation types. Queries in GraphQL are similar to how a GET request functions in REST where the query operation type will be used to query the GraphQL server. In a GraphQL query, developers are able to query specific fields based on the schema definition.

    Consider a situation where a developer wants to request for only the title field of posts from the JSONPlaceholder posts endpoints, the query can be written like this:

    // Mock GraphQL query
query {
 getPostTitles {
  title
 }
}
    GraphQL

    When we make a REST API GET request, all of the fields of the posts object are returned, however, when we execute this GraphQL query, only the post title field is returned thus preventing any over fetching of data.

    Mutations, on the other hand, are not used for fetching data but more for creating or modifying server-side data. Mutations are often used for state changing activities like DELETE, PUT, POST operations.

    Consider a situation where a developer wants to add a new post to the posts object, the mutation can be written like this: 

    // Mock GraphQL mutation operation 
mutation {
 addPost (
  title: “GraphQL Fundamentals”,
  body: “Learn how to write a mutation operation  
) {
  title
  body
 }
}
    GraphQL

    By running this addNewPost mutation, a new post will be added with the defined title and body variables shown above. Mutations, like queries, return data, but mutations only return newly created data. We defined the title and body fields in the mock mutation operation, and the title and body fields will be returned upon successful post creation. You can read up more on queries and mutation here.

    Lastly, let's look at resolver functions. A resolver is a function that is responsible for populating the data for a single field in your schema and it can populate the data in any way that you can define through functions.

    A resolver function generally accepts 4 positional arguments which are namely the parents, args, context and info parameters. Often, in a mutation operation, we will pass in parameters through the args variable to instantiate a new object.

    You can read up more on how to write query resolvers and mutation resolvers.

    Now that you understand how schemas, objects, and operation types interact with resolver functions, we will create a GraphQL API with Node.js and Apollo tying in all the above concepts.

  • Step 3: GraphQL Configuration

    In this step, we will be building a GraphQL API with Node.js and Apollo. Apollo is a Node.js library that provides an easy to use GraphQL server and it also comes with a user-friendly GraphQL interface for users to perform their queries and mutation operations during development.

    To get started, create a directory under your desktop following the steps below.

    cd Desktop
mkdir intro_to_graphql
cd intro_to_graphql

    Next, navigate to VSCode and open the intro_to_graphql folder created. Then, open up the terminal in VSCode and run the following command to initialise Node.js. If you do not have Node.js installed, you can head here to install and run the command to check if Node.js is installed. If Node.js is installed successfully, a version number will be returned.

    node -v

    With Node.js installed, run the following command to initialise the application.

    npm init -y

    Next, run the following command to install the necessary libraries.

    npm install apollo-server graphql

    Apollo-server makes it simple to set up a GraphQL server, and graphql is a required dependency for the Apollo-server. We are now ready to create your first GraphQL API!

  • Step 4: Build a GraphQL API with Node.js and Apollo

    In this step, we will create a hardcoded movies array and perform a mutation operation to add a new movie object to the array, followed by a query operation to list all of the movies in the array. First, create an index.js file in the root directory and this file will act as the entrypoint to the apollo server.

    Next, we will need to import the ApolloServer and gql libraries from the apollo-server node module previously installed. Copy the following code on line 1 of index.js.

    const { ApolloServer, gql } = require("apollo-server");
    JavaScript

    Next, create an array called movies where each movie will have a movieTitle and dateOfRelease field. Copy the following code on line 3 of index.js.

    // Create movies array
const movies = [
 {
   movieTitle: “Avengers - EndGame”,
   dateOfRelease: “Dec 22”,
 }
];
    JavaScript

    Next, we will need to initialise the typeDefs variable and in this variable, we will populate all the object and operations types. Copy the following code on line 11 of index.js.

    // Initialize type definitions 
const typeDefs = gql`
  type Movie {
    movieTitle: String!
    dateOfRelease: String!
  }

  type Query {
    queryMovies: [Movie]!
  }

  type Mutation {
    addMovie(movieTitle: String!, dateOfRelease: String!): Movie!
  }  
`;
    JavaScript

    The gql tag is used to convert the type definitions into GraphQL strings that can be read by the Apollo Server. In the typeDefs variable, we have defined the Movie type with the movieTitle and dateOfRelease fields of string type.

    Next, we also defined the query type that expects to return the movies array when executed. Lastly, for the mutation type, we created the addMovie mutation which takes in movieTitle and the dateOfRelease fields as parameters and expects a new movie type object to be returned when executed.

    Next, we will write the resolvers functions. The resolvers should match the typeDefs variable defined. Copy the following code on line 27 of index.js.

    // Create resolver functions
const resolvers = {
  Query: {
     queryMovies: () => movies,
  },

  Mutation: {
     addMovie: (parent, args) => {
       movies.push(args);
       return args;
   },
 },
};
    JavaScript

    The queryMovies function when invoked will return the movies array. For the addMovie function, it will take in two arguments which are the movieTitle and dateOfRelease of type string and return the newly created movie with the defined arguments. The newly created movie will also be populated into the existing movies array.

    Next, with the type definitions and resolver functions defined, we will now initialise the GraphQL server. Copy the following code on line 41 of index.js.

    // Initialize GraphQL server
const server = new ApolloServer({
  typeDefs,
  resolvers,
});

server
  .listen({ port: 8080 })
  .then(({ url }) => console.log(`GraphQL server running at ${url}`));
    JavaScript

    In this code snippet, we have created a new instance of apollo server and passed in the typeDefs and resolver functions we created in the earlier steps. Then, we initialize the GraphQL server on port 8080!

  • Step 5: Run the GraphQL API

    In this step, we will run the GraphQL API on the apollo server. On your terminal, run the following code to initialise the server.

    node index.js

    You should see the console log of “GraphQL server running at http://localhost:8080” indicating that the server is booted up successfully.

    Next, navigate to localhost:8080 and click on the “Query your server” button to navigate to the GraphQL playground as shown below. The GraphQL playground allows us to test our GraphQL API by performing queries and mutation operations directly on the server without connecting to a frontend.

    We will first run the queryMovies function. Paste the following code into the workspace.

    query {
 queryMovies {
  movieTitle
  dateOfRelease
 }
}
    GraphQL

    Next, click on the “Run” button to execute the queryMovies function and you should see the movies array output. The movies array is the hardcoded array we have written in index.js.

    We will now execute the addMovie mutation to add a new movie to the array. First, create a new workspace by clicking on the “+” icon. Next, paste the following code on a new workspace.

    mutation {
 addMovie(
  movieTitle: "Avengers - Kang Dynasty",
  dateOfRelease: "Dec 24"
 ) {
  movieTitle
  dateOfRelease
 }
}
    GraphQL

    Note: You are free to use any movie title and date of release for the new movie!

    Click on the “Run” button to execute the addMovie mutation and you should see the movieTitle and dateOfRelease returned as shown. This indicates that the mutation is successfully executed and the new movie has also been added into the movies array.

    To verify if the new movie has been added to the movies array, navigate to the query function workspace and run the query function again. You should now see that there are 2 movies returned as shown below.

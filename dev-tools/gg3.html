Git Online with GitHub

3 steps

In the previous tutorial, you opened the command line and practiced using Git locally. In this tutorial, we will be teaching you how to take your local repository online.

This makes your repository available to others who may not have access to your local machine. It is a crucial step in pursuing software development work with other developers.

Helpful prior knowledge

  • There are no pre-requisites

Learning Outcomes

By the end of this tutorial, players will have:

  • Created a GitHub account
  • Created a new repository
  • Added a local repository to GitHub

Tutorial Steps

Total steps: 3

  • Step 1: Creating a GitHub Account

    In the previous tutorial, you created a local repository. If you're not sure where it is, locate that repository for this tutorial. It's time for us to put it online!

    While Git allows us to do version control, GitHub very conveniently allows us to host our projects online and helps us collaborate with other developers.

    The repository you created in the previous tutorial may not be the fancy source code that impresses your future boss or that other collaborators want to work off, but the whole point of this exercise to learn version control and how to use Git and GitHub so that you can share your portfolio and start working with others as an aspiring developer.

    Once you’re ready, go create a GitHub account!

    Despite its popularity, GitHub is certainly not the only collaborative solution available. Below is a list with some alternatives that you may explore in your leisure. If you find that they work better than GitHub for you, feel free to use those instead.

    🔗 Article: GitHub Alternatives

    Do note that the instructions for this quest will pertain to GitHub for the sake of simplicity and consistency.

  • Step 2: Adding a Local Repository onto GitHub

    Once you’ve created your GitHub account, let's upload your repository. Here’s where things can get a little complicated.

    For a start, follow the instructions on GitHub below. Name your new repository yourstackupname_gittutorial, replacing yourstackupname with your username. Set your repository to "Public" if you would like others to have access to it.

  • Step 3: My Code Won’t Push! <Independent Problem-Solving>

    You may find, after attempting the final step in the instructions above, that you were not able to push your code to GitHub. One of the most important skills of being a developer is the ability to find solutions to problems when they crop up.

    Since we at StackUp want to help you develop this crucial skill, we won’t be providing the solutions to your problems directly. However, we will give you some hints to point you in the right direction.

    If you selected HTTPS in Step 7

    You will likely be asked for authentication. This is expected.

    Note: the HTTPS method requires you to provide your username and password at each visit.

    1. If you entered your GitHub username and password, you will be told that GitHub now requires you to use a Personal Access Token in place of your GitHub password.
    2. Google how to set up your Personal Access Token. Follow the instructions.
    3. Try to push your code again.

    If you selected SSH in Step 7

    You will likely be told that the system could not read from remote repository. This is expected.

    Note: The SSH method requires you to add your SSH key to your GitHub account to use it for authentication. You may or may not already have an SSH key.

    1. Google how find out if you have an SSH key. If you have an existing SSH key, skip the next step.
    2. If you do not have an existing SSH key, google how to generate an SSH key. Follow the instructions to generate an SSH key.
    3. Google how to add an SSH key to your GitHub account. Follow the instructions.
    4. Try to push your code again.

    If successful, you should see the image in Step 2.

Tutorial Steps

  • Step 1: What is Docker Compose?

    Docker has revolutionized the way we package and deploy applications!

    But what happens when our application stack grows and becomes more complex with services like Node.js, MongoDB, Ruby on Rails, RabbitMQ, and more? Running each service in its own container means we'd have to pull and build multiple images individually, followed by running them individually as Docker containers.

    The complexity increases further when these services have dependencies among each other, necessitating them to be started in a specific sequence. Manually managing such a setup can quickly become cumbersome.

    To overcome this, Docker introduced Docker Compose!

    What is Docker Compose?

    Docker Compose is a tool that allows developers to define and share multi-container applications.

    For example, assume that youre building a project with NodeJS and MongoDB together, you can now create a single image that starts both containers as a service, eliminating the need to start each separately!

    Benefits of Docker Compose

    • Simplified Configuration - Docker Compose uses a declarative YAML file to define and configure your application services. This file becomes the blueprint for your application stack, allowing anyone to recreate your environment consistently!
    • Efficient Development Workflow - Developers can easily start, stop and rebuild services with a single command allowing you to set up your entire application infrastructure with ease!
    • Dependency Management - Ensure services start in the correct order based on application dependencies!

    With a better understanding of Docker Compose, in the next step, we will be installing Docker Compose on our terminal and explore some of the key syntax of Docker Compose.

  • Step 2: Docker Compose Installation and Syntax Walkthrough

    In this step, we will install Docker Compose onto our terminal and walk you through the key syntax to build a Docker Compose file!

    To check if you have Docker Compose installed, run the following command.

    docker compose version

    You should observe a version number on your terminal output indicating that Docker Compose is installed on your device as shown below.

    Otherwise, you can head over here to install Docker Compose!

    With Docker Compose installed, we will next explore some of the key syntax of a Docker Compose file.

    Key Syntax of Docker Compose 

    • version - Specifies the version of the Docker Compose file
    • services - Defines each service (Docker Container) that should be run in your application
    • build - Provides build instructions if youre not using a pre-existing image from Docker Hub
    • image - Specifies the Docker image to use for a service (Docker Container)
    • ports - Maps ports from the host to the service
    • volumes - Defines any storage volumes you would like to attach to the service
    • depends_on - Define the dependencies between services in your application
    • networks - Specifies which networks the service is connected to

    To better illustrate these concepts, heres a sample Docker compose file!

    version: '3'

services:
 web:
  image: nginx:latest
  ports:
   - "8080:80"
  depends_on:
   - redis
  networks:
   - webnet

 redis:
  image: redis:alpine
  networks:
   - webnet

networks:
 webnet:
    Docker

    Summary of Code Snippet

    • We are using version 3 in this Docker Compose file format
    • There are 2 services in this application namely web and redis
    • For the web service, we use the nginx image and map port 8080 on our host to port 80 on the Docker container. We also specify a dependency on the redis service ensuring that the redis service starts before web. Lastly, the service is connected to the webnet network
    • For the redis service, we use the redis image and this service also uses the webnet network.
    • webnet is a custom network that we have defined that allows the web and redis services to communicate with each other

    With a better understanding of the key syntax of Docker Compose, we will run a multi-container application with Docker Compose in the next step!

  • Step 3: Create Application

    In this step, we will be building a multi-container application with Docker Compose!

    To begin, head over to the root directory and run the following code to create a new project folder!

    mkdir intro_docker_compose

    Next, run the following code to create the necessary working files under the intro_docker_compose directory!

    touch app.py compose.yml Dockerfile requirements.txt

    After running the following command, you should have the following file structure under the project directory!

    Next, copy the following code snippet and paste it on app.py!

    import time
import redis
from flask import Flask

app = Flask(__name__)
cache = redis.Redis(host='redis', port=6379)

def get_hit_count():
  retries = 5
  while True:
    try:
      return cache.incr('hits')
    except redis.exceptions.ConnectionError as exc:
      if retries == 0:
        raise exc
      retries -= 1
      time.sleep(0.5)

@app.route('/')
def hello():
  count = get_hit_count()
  return 'Hello World! I have been seen {} times.\n'.format(count)
    Python

    In this code, we will set up a web application using Flask that keeps track of the number of times the main endpoint is accessed, leveraging Redis as a storage backend on its default port 6379.

    When this Flask application run, whenever a user access the root URL , the hit count in Redis will increment and will display a message indicating how many times the endpoint has been accessed under the get_hit_count() function!

    Next, copy the following code snippet and paste it on requirements.txt!

    flask
redis

    These will be the libraries that are required to run our multi-container application!

    Next, proceed to copy and paste the following code snippet to the Dockerfile!

    FROM python:3.7-alpine
WORKDIR /code
ENV FLASK_APP=app.py
ENV FLASK_RUN_HOST=0.0.0.0
RUN apk add --no-cache gcc musl-dev linux-headers
COPY requirements.txt requirements.txt
RUN pip install -r requirements.txt
EXPOSE 5000
COPY . .
CMD ["flask", "run"]
    Docker

    Key Takeaways of Dockerfile

    • FROM python:3.7-alpine sets the base image for the Docker container
    • WORKDIR /code sets the working directory inside the Docker container to be /code
    • ENV FLASK_APP=app.py sets an environment variable FLASK_APP to app.py
    • ENV FLASK_RUN_HOST=0.0.0.0 sets the FLASK server to listen on all public IPs to the container

    In summary, the Dockerfile creates an image for the Flask application, sets up the necessary environment in the Docker container, installs the necessary dependencies and runs the Flask application as a Docker container!

    Lastly, we will be populating the compose.yml file!

    Copy the following code snippet and paste it to your compose.yml file.

    services:
 web:
  build: .
  ports:
   - "8000:5000"
 redis:
  image: "redis:alpine"
    YAML

    In this compose file, we define 2 services namely: web and redis.

    The web service is built using the Dockerfile located in the current directory. It maps the containers port to the host machines port 8000!

    The redis service uses a public Redis image pulled from the Docker Hub registry.

    With that, we are ready to run the multi-container application in the next step!

  • Step 4: Run Application

    In this step, we will be running the multi-container application that we have prepared in Step 3!

    To run the application, start by running the following command under your root directory.

    docker compose up --detach

    After running the command, you should observe the following on your terminal output as shown below!

    Running docker compose up first pulls the redis image from the Docker hub registry.

    Next, it will begin building the custom Docker image from your Dockerfile and once both images are ready, they will run both docker containers as services as defined under the compose.yml file!

    To verify if you have run the application correctly, head over to localhost:8000 and you should observe the following as shown below.

    You should notice upon every refresh that the value increments by 1!

    We have successfully run a multi-container application built using Flask and Redis with Docker Compose!

    You can proceed to run the following command to stop the application!

    docker compose down

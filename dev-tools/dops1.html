Introduction to GitHub Desktop

3 steps

The first tutorial in this GitHub DevOps module will introduce players to GitHub Desktop as a GUI client for Git. This tutorial will revise the basic Git features as part of the DevOps process and serve as a tour for GitHub Desktop.
Join us on Discord

Helpful prior knowledge

  • There are no pre-requisites

Learning Outcomes

By the end of this tutorial, you will be able to do the following with GitHub Desktop:
﻿

  • Create a repository
  • Create branches
  • Commit and push to branches
  • View commit history and comparisons

Tutorial Steps

Total steps: 3

  • Step 1: Installing GitHub Desktop

    GitHub Desktop is a GUI client for GitHub which enables users to make use of Git and other collaborative features of GitHub. GitHub Desktop is for users who prefer using a graphical user interface over a command line interface like the Windows terminal, providing ease of access to the features of Git without having to memorize and write commands.

    IMPORTANT

    This tutorial builds on previous knowledge about version control, Git and GitHub. If you have not attempted our previous module on Version Control with Git and GitHub or cannot answer the following questions, we recommend that you do some revision with the previous module first.

    Checklist for Understanding﻿

    1. What is version control? Who can benefit from using version control systems?
    2. Is there a difference between Git and GitHub? If so, what are the differences?
    3. What is branching in Git? When would a developer create a branch and merge it?
    4. What is a commit?
    5. What does it mean to push changes to GitHub?

    After you’ve made sure that you understood the above, you may proceed with this quest.

    Downloading GitHub Desktop

    If you are using a Windows computer, just head over to GitHub Desktop’s website and click the “Download for Windows” button. Then, run the executable and follow the installation guide.

    For Mac users, go to the same website and click the “Download for macOS” button. Unzip the zip folder downloaded, and drag the file with the “.app” extension into your Applications folder.

    Although GitHub Desktop does not support Linux directly, there are forks of it and tutorials to install it available online, such as this GitHub gist. You may follow the guide on Linux distributions such as Ubuntu and Kali Linux as mentioned by users who have managed to get it working.

  • Step 2: Creating a Repository

    Now that you have GitHub Desktop installed, it’s time to create your first repository. Open GitHub Desktop, sign into your GitHub account and complete basic setup.

    On the top left corner of your desktop, click “File”, then “New repository”. Here, a window will pop up, prompting you to fill in the repository name, description, local path and more. It should look like the screen below.

    For this tutorial, name the repository “Username-Intro to GitHub Desktop”. You should replace “Username” with your StackUp username. E.g. if your username is Stackie1234, your repository should be named Stackie1234-Intro to GitHub Desktop.

    Then, set the description to “DevOps with GitHub - Part 1”. Set the local path to a directory of your choice–this is where your repository folder will be stored on your device.

    For the rest of the settings, you may use the default values. Once done, click “Create Repository”.

    Next, click “Publish Repository” at the top right corner of GitHub Desktop to initialize your repository on GitHub. Once done, your window should look something like this.

    If you head to your GitHub account using your browser now, you will see that Username-Intro to GitHub Desktop has been added to your repositories. However, if you enter the repository, you will notice that it is currently empty. This is because test.txt, which is on your local machine, has not been pushed to GitHub yet!

    To do this, from the GitHub Desktop application, click the “Commit to main” button, then the “Push origin” button at the top right, to push your updates to GitHub. Now refresh your repository page in your browser and you should see your new file.

    Now, we want to create a second branch to edit the files without altering the main branch. Click the “Current branch” button and subsequently the “New branch” button. A prompt will appear asking you for the branch name, name it “test-branch”. Then create the branch and publish it to GitHub. Notice that your current branch is now set to “test-branch”.

    Lastly, we want to view our commit history to analyze changes made to the files in each commit. Click the “History” tab and select “main” in the “Select branch to compare” field. Then click the “Ahead” tab as your branch “test-branch” is currently ahead of the main branch.

  • Step 1: Primer on Docker

    If you are new to Docker, this quest is specifically designed to equip you with the fundamentals of Docker before moving on to Docker Compose!

    Let’s begin with a primer!

    Origins of Docker

    Imagine stepping into a new company and being handed your first project. Before diving into the actual work, you're tasked with setting up your development environment. This means manually installing each dependency and library, meticulously ensuring nothing goes awry during this setup phase. Quite the daunting start, wouldn't you say?

    This is where Docker comes in handy!

    What is Docker?

    Docker is a platform designed to make it easier to create, deploy and run applications by using containers!

    You can think of containers as lightweight, stand-alone packages that contain everything an application needs to run the code such as the code, runtime, libraries and dependencies all configured.

    This ensures that the application will always run the same, no matter where it’s being deployed, effectively eliminating the “It works on mine, but it does not work on yours problem!”

    Benefits of Docker

    • Consistency: Applications run the same no matter where the container is deployed
    • Isolation: Containers run in isolation and do not interfere with each other
    • Portability: Containers can be moved easily and deployed anywhere
    • Microservices: Application can be broken down into microservices where each service can be run as a Docker container

    Docker Images versus Docker Containers

    • A Docker image is a lightweight, stand-alone package that includes everything needed to run the application including its code, libraries and configuration. Think of it as a blueprint.
    • A Docker container, on the other hand, is a running instance of a Docker image where if the image is the blueprint, you can think of the container as the house built from that blueprint.

    Use Cases of Docker

    • Developers can write code locally and share their work using Docker Images
    • Docker Containers can be used to simulate different environments such as in staging or in production where tests can executed automatically

    Now that you have a better understanding of what Docker is, up next, we will be installing Docker if you have not done so already!

    You can run the following command on your terminal to check if Docker is currently installed on your machine.

    docker -v

    If a version number is shown on your terminal, it means that Docker is successfully installed. Otherwise you can follow the instructions below to install Docker!

    Docker Installation

Important: Note that the installation executable differs depending on whether your Mac is currently powered by an Intel chip or an Apple Silicon chip.
    • For Linux users, head over here and follow the installation guide to install Docker on your Linux device

    Once docker is successfully installed, open your terminal and run the following command.

    docker -v

    Your terminal output should look like the following representing a successful installation of Docker on your device!

  • Step 2: Build Docker Image

    Now that we have Docker successfully installed on our device, up next, we will be running a Docker container in this step!

    To begin, run the following command to initialise a project folder on your desired directory.

    mkdir docker_compose_campaign

    Next, under the docker_compose_campaign directory, run the following command where we will be cloning a simple todo application repository.

    git clone https://github.com/docker/getting-started-app.git

    Next, open up the getting-started-app folder on your desired IDE where you should have the following file structure.

    Next, we will need to create a Dockerfile!

    A Dockerfile is a script that Docker uses to create an image. It starts by designating a base image, then details the environment variables, files to be copied, and commands to execute. Docker then uses this script to build a container image.

    To create a Dockerfile, under the root directory, run the following command.

    touch Dockerfile

    Next, copy the following code snippet to the newly created Dockerfile.

    FROM node:18-alpine
WORKDIR /app
COPY . .
RUN npm install
CMD ["node", "src/index.js"]
EXPOSE 3000
    Docker

    Summary of Dockerfile

    • FROM node:18-alpine specifies the base image to be used and in this instance, the image we are using is an Alpine-based image of Node.js version 18
    • WORKDIR /app sets the working directory inside the Docker container to /app where any subsequent command will be executed from this directory
    • COPY . . copies the current directory contents on the host machine to the current directory set inside the container (/app)
    • npm install installs all the libraries and dependencies defined from the package.json file
    • CMD [“node”, “src/index.js”] runs the index.js file located in the src directory
    • EXPOSE 3000 informs Docker that the container listens on port 3000 at runtime

    With a better understanding of the Dockerfile, we are now ready to build the Docker image!

    Run the following command to build the Docker image.

    docker build -t my_docker_image .

    The docker build command uses the Dockerfile to build a new Docker image. The -t flag tags your docker image and in our instance, we have tagged the image as “my_docker_image” for better readability!

    Lastly, the . at the end of the build command informs Docker that it should look for the Dockerfile in the current directory.

    Once you have successfully built your Docker image, run the following command to list all images on your terminal.

    docker images

    You should observe the following on your terminal indicating that your Docker image has been built successfully!

  • Step 3: Run and Stop Docker Container

    Now that we have a Docker image, we are ready to run our Docker container!

    Proceed to run the following command and specify the name of the image you just created.

    docker run -dp 127.0.0.1:3000:3000 <YOUR_DOCKER_IMAGE>

    ⚠️ IMPORTANT: Do ensure that you replace the placeholder with your docker image name!

    The -d in short for detach runs the container in the background.

    The -p creates a port mapping, publishing the container’s port 3000 to our localhost:3000 allowing us to access the application from our localhost!

    To verify that the docker container is run correctly, head over to localhost:3000. You should observe the following on your localhost indicating that your docker container is run successfully!

    Feel free to populate as many items as you wish!

    To view your Docker containers via the CLI, you can also run the following command.

    docker ps

    You should observe that the command returns your current running docker container!

    Next, we will be stopping and removing the Docker container. We will first stop the Docker container!

    Proceed to run the following command

    docker stop <CONTAINER ID> && docker rm <CONTAINER ID>

    ⚠️ IMPORTANT: Do ensure that you replace the placeholder with your Docker container ID! 

    Once these commands are run successfully, you should observe that the Docker container is successfully removed after running the docker ps command!

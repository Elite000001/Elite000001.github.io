Practicing the Command Line

6 steps

In the first tutorial, you made your first foray into interacting with your system using a command line interface. In the previous one, you learnt more about the hierarchical directory structure and how to navigate within it. You also practiced doing this in your own computer system. In this final tutorial of this module, you will deepen your understanding of the command line by learning about and practicing more commands.

Some of the resources linked in this tutorial are quite comprehensive. We recommend that you study and try out the commands in your own time. You need not understand all the parameters of each command to be able to complete the exercise at the end of this tutorial. However, we do recommend that you continue to explore each of these commands even after completing the exercise.
Join us on Discord

Helpful prior knowledge

  • There are no pre-requisites

Learning Outcomes

By the end of this tutorial, players will be able to:

  • Execute basic commands using a CLI independently, including performing the following:
  • Using wildcards to search for files
  • Combining commands with pipes
  • Searching for text with grep
  • Transferring data to or from a server using curl

Tutorial Steps

Total steps: 6

  • Step 1: (Revision) The Very Basics

    This step is revision-oriented. If you've forgotten what was covered in the past 2 tutorials, do go back to try them out again. We want you to be confident and comfortable with executing the commands we previously covered before adding these slightly more complex ones to your command line tool set.

    Before moving on from this step, make sure you are familiar with the following commands and what they do:

    • whoami
    • pwd
    • man
    • ls
    • cd
    • mkdir
    • touch

    Ensure also that you understand hierarchical directory structure and how to navigate directories using relative and absolute file paths.

    Once you're done revising, we'll teach you a few more concepts and commands that are commonly used on CLIs. We recommend that you open a CLI on your system to practice the next few steps with files on your computer.

    Please note that Windows users should be running a Linux distribution for this module. If you are a Windows user, and you have not installed WSL on your system, please follow the instructions in the tutorial "Introducing the Command Line" to do so.

  • Step 2: Using Wildcards to Search for Files

    When searching for files on your computer, you can use wildcards to help you. Wildcards are essentially symbols, such as * and ?. You use these symbols to represent unknown characters or match particular character sets.

    For example, you may be a student looking for an essay you have written. You cannot remember where you saved the file or the filename exactly because you wrote the essay at 3 in the morning. However, you remember that you added the word "final" to the name because this is the last edited version of the file. You could use * to represent the other parts of the filename and search for *final.

    Read the following articles to find out more about the uses of wildcards. 

     Article: What is a wildcard, and how can I use it?

     Article: 10 Practical Examples Using Wildcards to Match Filenames in Linux

  • Step 3: Combining Commands with Pipes

    Thus far, you have only practiced executing commands one at a time. While there's nothing wrong with doing things step-by-step, you'll soon find this inefficient and limiting. This is where pipes come in.

    Pipes allow you to combine two or more commands in the same line. With pipes, the output of your first command is used as input for your second command and so on. Find out more about how to use pipes in the article below.

     Article: Pipes and Filters in Linux/Unix

    Then, watch this video to see pipes in action.

  • Step 4: Searching for Text

    One of the most useful commands, grep allows you to search inside files for patterns of text. You may be wondering what the big deal is, since you could simply enter the file with your word editor or spreadsheet program and use the in-built search function to perform the search.

    Imagine having a spreadsheet with 1 000 000 rows of data. Simply opening the file using the spreadsheet program would take your system some time. grep doesn't require you to access the file with your spreadsheet program.

    Learn the grep command and its parameters in the resource below.

     Article: grep command in Unix/Linux

  • Step 5: Transferring Data to or from a Server

    Another useful command you may already have come across in some of our learning material is curl. curl facilitates the transfer of data to a client or from a server without need for user interaction. The curl command is natively available in Linux/Unix-based shells.

    Read up about how to use curl below.

     Article: curl command in Linux with Examples

    For more examples on how to use curl,

     Article: cURL Command Tutorial with Examples

  • Step 6: Exercise: Working with the Command Line

    Now that you've learnt how to do a few more things using a CLI, it's time for you to practice with an exercise.

    Follow the instructions below, then check your practice against our suggested answer in "Expected Output". Do your best to complete all parts through the CLI, since using the CLI is the purpose of this tutorial. You may choose to use one command to perform multiple parts.

    Open the CLI you've been practicing with, then complete the following steps:

    1. Print the current directory.
    2. In this directory, create a new directory named stackup_cli.
    3. Navigate to the stackup_cli directory.
    4. In the stackup_cli directory, create another directory named bumble_research.
    5. Navigate to the bumble_research directory.
    6. In the bumble_research directory, create a new file named readme.txt.
    7. In the directory, download the file located at https://firebasestorage.googleapis.com/v0/b/gitbook-x-prod.appspot.com/o/spaces%2FD2U7sZ458eU4PHVnX907%2Fuploads%2FtPNScvADCcQ6DCv9kzt9%2Fbumble_google_play_reviews_edited.csv?alt=media&token=bcaabe60-b472-4a48-9216-98c19185386f
    8. The above is a CSV file containing Google Play reviews of the dating app Bumble. For simplicity, this file has been reduced to 2 columns containing 1) the username of the reviewer and 2) their text review. The original can be found here.
    9. Change the name of the CSV file to bumble_data.csv.
    10. Let's say we work for Bumble and want to see how many times users compare our app to a competitorTinder. Count the number of rows in which the word "Tinder" appears in the file. Ensure that you
    11. include reviews where users do not capitalize "Tinder" (hint: case sensitivity)
    12. exclude reviews where the word "tinder" is part of another word, e.g. "tindersticks" (hint: whole words)
    13. The file contains more than 100 000 rows of data, which may make operating it very slow. Extract only the rows that fit the requirements of Step 10 and put them into a different file named tinder_bumble_comparison.csv. This file should be in the bumble_research directory.
    14. Now, count the number of rows in which the word "Tinder" (with the same parameters in step 10) appears in tinder_bumble_comparison.csv.
    15. Ensure you are still in the bumble_research directory. If not, navigate to it.
    16. Search for and list all files in the bumble_research directory with file names containing the word "bumble".
    17. Finally, using a relative path, navigate back out to the stackup_cli directory.

    And you're done! Check your practice against the "Expected Output" section for suggested commands to completing the steps above. Note that our suggested commands may not be the only way to complete each of the steps above.

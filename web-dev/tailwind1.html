Getting Started with Tailwind CSS

8 steps

In this tutorial, you will learn what Tailwind CSS is and its advantages in comparison to other CSS frameworks. The utility classes are a core Tailwind CSS feature. You will learn how they work and why it is a good choice to use the utility-first workflow as a web developer.

In addition, you will learn how to install and set up Tailwind CSS in a project. Furthermore, you will be introduced to an extension and a plugin to assist you in remembering all the utility classes and avoid thinking about sorting them in the HTML. By the end of this tutorial, you will check whether your Tailwind CSS setup is correct and what a project built using this framework looks like.
Join us on Discord

Helpful prior knowledge

CSS Basics, Intermediate

Learning Outcomes

By the end of this tutorial, you will be able to:

  1. Explain Tailwind CSS and its difference from other popular CSS frameworks
  2. Describe how utility classes work and why to use them
  3. Install Tailwind CSS in any project with node package manager and set it up correctly

Tutorial Steps

Total steps: 8

  • Step 1: Discovering Tailwind CSS

    Let's start our introduction to Tailwind CSS by watching the video above, to get a good overview of what this framework is.

    Tailwind is an efficient CSS framework based on a functional approach. With Tailwind CSS, you dont actually write CSS code. Instead, you type pre-styled classes in your HTML file which are also known as utility classes.

    Also, Tailwind CSS works by scanning all of your HTML files, JavaScript components and any other templates for utility classes, generating the corresponding styles and then writing them in a static CSS file so that the browser can actually run your code.

  • Step 2: Understanding the Utility Classes

    As you already know, Tailwind CSS is based on using the utility classes directly in your HTML file to style elements. Now, let's understand why this approach is efficient to build websites quickly and how it helps you design them.

    Article: Utility-First Fundamentals - Tailwind CSS

    In this article you will learn about:

    • Using a conventional approach with CSS versus using utility classes to code custom designs
    • How utility classes are different from inline styles
    • How Tailwind CSS solves the biggest maintainability concern when using a utility-first workflow
  • Step 3: Installing Node.js

    In this tutorial, we will install Tailwind CSS through the Tailwind command line interface (CLI) tool, which is the fastest method to start using Tailwind CSS from scratch. However, before starting the installation we need to install Node.js.

    To run Tailwind, we will need the latest version of Node.js. In the event you already have a version of Node.js installed, we can install Node with the Node Version Manager (NVM), which will make it possible to install multiple versions of Node.js on your computer. Check out the instructions in this guide to install NVM. Be sure to download the setup file for v1.1.11.

    Once NVM is installed, run the following command in your terminal to install the latest version of Node.js.

    Once NVM is installed, run the following command in your terminal to install the latest version of Node.js. 

    nvm install latest

    In the event that you have another Node version, run the following command in your terminal (preferably with administrative privileges) to use the latest version. At the time of publication, the latest version is 20.1.0.

    nvm use 20.1.0

    With the setup complete, let's check that you are now using the latest version. Run the command below in your terminal.

    node -v

    Your output should display the current version of Node.js that your system is using.

    In the event that you see an output of panic: runtime error: index out of range [3] with length [3], ensure that your NVM version is 1.1.11.

  • Step 5: Installing Tailwind CSS through the Tailwind CLI Tool

    While working on your project, if you would like to view how your site changes, you may install a Visual Studio extension called Live Server by Ritwick Dey. This extension launches a development server which reloads your webpage each time you make changes to your code. In your top toolbar, go to File > Preferences > Extensions, and search for it. Alternatively, you can find it here. Once installed, open your index.html file and click on the Go Live button in the status bar, found at the bottom of your VS Code window.

    Notice that even with all the utility classes and with the CSS file linked, no styling has been applied. To solve this, it is necessary to install the Tailwind CSS in the project so that it can generate the styles of the utility classes into a file, set by you.

    Firstly, we will initialize npm on the project so that you can install the Tailwind CLI tool package into the project. Open up the terminal and make sure that you are in the project's root directory, that is, FrontStackies and enter the following command.

    npm -y init

    Your terminal should look like the image below and a package.json file should be created in your root directory.

    Now, you will install the tailwind CLI tool as a devDependency, which means that the Tailwind CLI tool is a package only required during the development. Enter the following command in your terminal.

    npm install -D tailwindcss

    Your terminal will load the installation of the packages. Now, open your package.json file and check that 'tailwindcss' appears as a devDependency, which you should find in lines 12 and 13.

    Now that the Tailwind CLI tool is installed, we can run it by entering the following command in your terminal.

    npx tailwindcss init

    Notice that the tailwind.config.js file has been created in FrontStackies.

  • Step 6: Configuring Tailwind CSS

    The tailwind.config.js file is responsible for several Tailwind CSS configurations. As this tutorial introduces you to Tailwind, we will only configure the "content", which informs Tailwind CSS where to look for the files that will generate the corresponding styles.

    In your tailwind.config.js file, update the content property in line 3 with the following code.

    content: ["./src/**/*.{html,js}"],
    JavaScript

    Finally, you need to set how Tailwind will find its utility classes. Enter the following code in the input.css file. We are using the @tailwind directive to inject Tailwind's 'base', 'components' and 'utilities' styles.

    @tailwind base;
@tailwind components;
@tailwind utilities;
    CSS

    You can refer to the image to check if you have updated the two files correctly. 

  • Step 7: Building Your CSS with Tailwind CSS

    Now that you have Tailwind CSS installed and configured correctly, it's time to get working!

    In index.html you can see several utility classes being used. However, notice that in line 6, we are linking to output.css which has not been created yet. This file will be created by the Tailwind CLI tool during the CSS build process when Tailwind CSS generates the utility class styles into the static file output.css.

    To build the CSS, run the command below in your terminal and leave your terminal open. We are using the watch option which starts a watch process. This means that for every change made, Tailwind will automatically rebuild the code.

    npx tailwindcss -i ./src/input.css -o ./dist/output.css --watch

    After that, when viewing your index.html file in the browser, you should see a web page just like the image below. Congratulations! You've completed your first project with Tailwind CSS!

Deploying Sites with Netlify

6 steps

Traditionally, applications that involve data transmission often necessitate their dedicated servers. However, the concept of serverless computing has transformed this landscape. It enables the creation of scalable applications that operate seamlessly across multiple machines, requiring minimal direct management. This is where Netlify comes in. With Netlify, your projects gain the advantage of unique domains, ensuring rapid and efficient deployment.

To fully leverage the capabilities of Netlify, we will start by familiarizing ourselves with its essential tools. In this tutorial, we will delve into the specifics of what makes Netlify stand out. We will explore its core features and functionalities and guide you through integrating its tools into your development workflow, paving the way for a more streamlined and effective web development process.
Join us on Discord

Learning Outcomes

By the end of this tutorial, you will be able to

  1. Discover the functionality of Netlify and the other services it offers
  2. Install Netlify tools on your local machine
  3. Deploy your site to Netlify

Tutorial Steps

Total steps: 6

  • Step 1: What is Netlify and Why Should I Learn It?

    Before we start installing the necessary tools needed for us to use Netlify, watch the video above. Netlify provides everything you need to take modern web projects from the first preview to full production. This includes features such as automatic builds, deploys, and global edge networks. Netlify integrates with any other hosted service via webhooks. This allows you to trigger new deployments whenever an event occurs.

    Next, read this article on Netlify. This article outlines what Netlify is and why it is preferred by the authors. It also details the benefits of using Netlify for both developers and editors. Some of the important points from this website are that Netlify is a web hosting infrastructure and automation company. Netlify is also very easy to use and can be used by developers of all levels.

    Netlify and how it works

  • Step 2: Creating An Account With Netlify

    We will need to first create an account with Netlify. You can do so here.

    You can choose to sign up using your GitHub account or email address. Also, do note that it does not cost anything to create an account.

    In fact, upon signing up, you will be on the Starter Plan which is free. It provides you with 100 GB of bandwidth each month, 300 build minutes and up to 500 sites. If you wish to take your developer journey with Netlify, you can consider subscribing to some of their paid plans.

  • Step 3: Installation of Node JS

    In this step, we will install the tools we need for Netlify. Firstly, we need to install a JavaScript Package manager called Node. This will allow us to install Firebase Tools onto our machine. Head here to install Node onto your machine.

    Select the installer for the machines operating system. It will install an executable installer for seamless installation. Run the installer.

    Once you have opened the installer, follow the on-screen steps to safely install the file onto your machine. Node will then install some packages for you to install dependencies for any JavaScript project you may have.

  • Step 4: Netlify CLI

    Finally, we are almost done with setting up our environment for Netlify. We will need to install the final component which is the Netlify command line interface (CLI). For us to upload files, deploy files for hosting, and initialise the project as a Netlify, we will need to do so via the CLI. This is the same for all operating systems.

    Open your native terminal or command prompt and enter this command.

    npm install netlify-cli -g

    Do remember this command as it is the standard command used for all JavaScript projects that require Nodes dependencies files. The first word: npm means Node Package Manager and for most command line tools, the first word is usually the program or app responsible for that command. The next word install is the command for that particular program. For this case, relevant packages will be downloaded and installed onto our machine. Next is -g which is known as a flag. Flags are additional commands which can be added to the current command function. Any packages installed in this will be stored in a global directory on your local machine. This means that any project that requires this package can refer to it without having to install it in the future.

    Once done with the installation process, you can enter the following command to check if Netlify has been successfully installed. A version number should be returned.

    netlify -V

    Some users may face privileged restrictions or errors. This is due to Node installing Netlify into a folder which requires administrator privileges. This can be solved by doing the following:

    For Windows users:

    Open Command Prompt as Administrator. Run the previous code, you should be able to run it now.

    For MacOS and Linux users:

    Type the following code into your terminal.

    sudo netlify -V

    Sudo stands for Super User which will prompt you to enter the administrators password to allow the necessary tools to run.

  • Step 5: Authentication

    Netlify CLI uses an access token to authenticate with Netlify. You can obtain this token using the command line. In your terminal, enter the following command.

    netlify login

    A new browser tab or window will be opened asking you to authorize Netlify CLI to access Netlify on your behalf.

    Click on the Authorize button. If the browser tab or window is not opened, you can also copy the URL in your terminal and input it in your browser. Once authorization has been granted, your terminal will show the message "You are now logged into your Netlify account!"

    A little more information about where the access token is stored. It is stored in a config.json global configuration file. When you use the Netlify CLI in future for deployments, the token in this file will be used. For Windows users, this file is stored in your AppData\Roaming\netlify\Config directory, whereas for macOS users, it is stored in Library/Preferences/netlify.

  • Step 6: Deploying Your Site

    We can now deploy our site manually. This method assumes that you have already used the npm run build command in your project. The command creates a build folder with the final product that can be used in production.

    In your terminal, enter the following command.

    netlify deploy

    You will come across the following prompts and these are the responses you should make:

    • What would you like to do?: Create & configure a new site
    • Team: <choose the default option>
    • Site name: <a name of your choosing>

    On asked to provide a publish directory, enter build. Once the files have been uploaded, you will be provided with a "Website Draft URL". This site is meant for previewing and testing. Go to that website and check that your website works as intended.

    If everything looks good, use the following command.

    netlify deploy --prod

    Once again, you will be asked for the publish directory. Use build.

    When you see the message "Deploy is live!" and your website URL is provided, you know that your site has been successfully deployed. Visit your website again and ensure that everything works well.

    With that, you have successfully completed this tutorial and are now able to deploy your web development projects to Netlify.

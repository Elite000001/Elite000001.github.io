Tutorial 1 - Introduction To Redux Toolkit

5 steps

In this quest, we will start by introducing you to the basics of Redux and Redox Toolkit, giving you some comparisons from a simple Counter app between using React's state management vs Redux Toolkit + Redux React's state management.

  • Manage states with Redux and Redux Toolkit

Tutorial Steps

Total steps: 5

  • Step 1: States in React

    States refer to the complete description of a system at a given time. In React, this usually means what happens in a web page such as the user's inputting on a form and then submitting it and in the background, a lot of state changes are happening from user input changing the state's field variables for the form data to sending data to the backend and sent back so that the application performs a state change i.e. log out to logged in state of a user.

    Let's try an example by initialising a React project assuming you already have NodeJS installed either through nvm or fnm or from the official installer. Head to https://nodejs.org/en/download/ and select your preferred installation. We recommend that you use an LTS version of Node. We require you to enable corepack during the installation process to enable yarn, the alternative project and package manager for node applications.

    Once done, head to https://github.com/uncomfyhalomacro/advanced-state-management-with-redux-toolkit-template. This is a custom personal vitejs template that uses react-swc + vitejs and uses the linter and formatter from https://biomejs.dev. Then click "Use this template" and then in the drop-down, click "Create new repository". You can name the repository in any way you want but for this tutorial, we recommend it to be advanced-state-management-with-redux-toolkit.

    Then you can either decide to locally clone the repository, or use GitHub Codespaces or Gitpod to initialise your development environment. The name of the new repository based on the template is up to you.

    We are going to use VSCode throughout the campaign. We advise you to install the Biome plugin for this quest just for the linting and formatting.

    🙂 Although we welcome you to use any other editor, we won't be able to help out that much if it's some configuration that we are not familiar with.

    Okay! Once ready, let's head to the root of the project directory where your package.json is located. Run the following command

    yarn install && yarn dev

    This should run a web page at http://localhost:5173. Open the URL on your browser and you should see a webpage that looks like the screenshot below

    Clicking the "count is 0" will increment the number by 1. If you are familiar with React, this should not be a surprise for you. But for explanatory purposes for those who aren't, let's read further down 😉

    Go to src/App.tsx. You can see these snippets of code

    // snipped
const [count, setCount] = useState(0)
// snipped

          count is {count}
        </button>
    TypeScript

    This app is basically a counter app. Every time a user clicks the button element, it will change the state of the count variable by incrementing its value by 1 (per click) using the useState hook from React. The name really fits what it does. It provides a function that is targeted to the count variable called setCount.

  • Step 2: Understanding State Management in React

    Our counter example is too simple to give a full picture of state management in React but still enough to give us an idea how it works. When handling states in React, these are done via React's useState, useEffect, and useContext (some are not mentioned such as useMemo). Here is an example for useState

    This is good and all but when trying to retrace your mutated states from multiple state variables for the entirety of your application caused by those hooks, it becomes cumbersome. Also, trying to move multiple states downwards or upwards between component trees can become unwieldy for the developer.

    Although it is possible to move state between components based on React's https://react.dev/learn/sharing-state-between-components#step-2-pass-hardcoded-data-from-the-common-parent, this results in increasing prop complexity when there are many props that need to be managed or receive state changes. Thus, this limits React's state management because it's per component and of its unidirectionality (parent to child i.e. Component 1 passes props to Component 2 but not the other way around).

    🤯 By the way, if you have encountered such problems where you have to do some hacks in React to manage state before knowing about Redux, feel free to share your experience in our forum channel.

    This does not mean that the state management hooks provided by React are bad, they're just limited on things like overall application state or global state. It's still sufficient for a per local component state.

  • Step 3: Easing Out State Management with Redux

    To solve the issue of global state management, Redux was created.

    The Redux documentation describes that Redux is "[a] JS library for predictable and maintainable global state management".

    What does it mean to have a predictable global state? It means that a developer knows that the state of their application at a specific action at a specific time is consistent with their definitions they made for their global state.

    This also means that whenever there is a state bug, that usually means that the state definitions that were made using Redux need to be fixed. But this is not an issue because the global state is centralised which also means there is one point of failure to look at and fix; whereas not using Redux will lead developers debugging each functional component of their web app leading to a lot of time to debug!

    This also means that using Redux makes your application easier to maintain AND debug! 👀

    But what are the cons of using Redux? Although Redux is a good library for managing global states, it's not always needed for everything (for example, we mentioned before that React's state management is sufficient in most cases). Another issue would be the increased boilerplate to define your application global states, which fortunately, is mitigated with Redux Toolkit. This is not a bad thing but something you should consider when you should be managing global states or not with Redux. This is well documented in "When should I use Redux" from the Redux documentation.

    Okay, now that's all covered, let's try experimenting with Redux in the next step.

  • Step 4: Experimenting Redux with React Redux

    The recommended way to use Redux now with React is to use the Redux Toolkit with React Redux. This avoids us from requiring too much boilerplate from Redux, the library, itself.

    Head back to your project and run the following command at the root of your project directory.

    yarn add --exact react-redux @reduxjs/toolkit
yarn add --dev --exact @types/react-redux

    This should add Redux and Typescript support for Redux with React as your dependencies. Feel free to copy the code from the source. They are explained below.

    In the new source code, we replaced the usage of useState from React to the one we have from Redux. You can check out the following links to the committed changes from useState to using Redux's state management hooks

    • App.tsx
    • src/app/hooks.ts
    • src/app/store.tsx
    • src/main.tsx#L5-L6

    Describing the Changes

    With Redux Toolkit + Redux React, we have overhauled the use of the useState hook from React.

    We have created the directory src/app relative to the project directory root. This contains two files: hooks.ts and store.ts.

    The store.ts file contains the configureStore at line 33, which is an abstraction of the createStore which is considered deprecated but still used internally — see https://redux.js.org/usage/migrating-to-modern-redux.

    It is designed to hold the global state of your application. Hence, we have to declare some of the following that might be new to you.

    To start, we need to define what is a reducer and an action. A reducer is a function that responds and mutates/creates new state caused by data sent by an action function to the store which is acted upon, for example, a user clicking a button.

    The diagram shown above is state management in Redux. There are actually 2 representations of a state not shown here — the original and current state. They will be discussed later on the next quest.

    The hooks.ts file contains the custom hooks called useAppSelector which is responsible for selecting a part of the Redux store's state or computing derived data and useDispatch which dispatches new data from an action called increment to be processed by the reducer as a new state.

    This new state change is then stored to the store which is then "read" by the application, therefore, the application performs a change caused by the state change.

    Overall, this might look like overengineered state management when we migrated away from React's useState for a counter app. However, Redux's state management is more flexible in that we can do state changes for the overall application and we are not limited by the component tree from React.

    This might not be visible now but with big projects that have multiple states and multiple states to track, Redux is the perfect toolkit for your state management as you can

    • Define the global state of your application
    • Create custom types to define your states leveraging Typescript's type system
    • Create logs for global state changes that are hairy to implement with React's state management hooks

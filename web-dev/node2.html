Views and SSR with EJS

7 steps

Note: This quest has been tested on Gitpod.io, a cloud development environment with a standardized set up. This means that your experience going through this quest will be the same as the quest master's – seamless!

In the realm of web development, many avenues exist for crafting your Front End. You have the freedom to choose from a variety of front-end frameworks such as NextJS, ReactJS, and more. These frameworks harness the power of Client Side Rendering (CSR) to construct engaging web interfaces for users.

In this journey, we will embark on a quest to master the creation of a Server Side Rendering (SSR) back-end application. Through SSR, we will serve the user-requested content directly from our server, eliminating the need for frameworks like NextJS or ReactJS.

Our tool of choice will be EJS, or Embedded JavaScript, a versatile templating language that empowers us to generate templates for the Front End. Leveraging HTML and CSS, EJS breathes life into the structure and aesthetics of our applications. EJS comes with a distinct file extension, ".ejs," which will seamlessly be handled by the EJS Middleware.

Without further delay, let's dive into the process of crafting our back-end application to seamlessly render EJS content for our users!

Helpful prior knowledge

JavaScript Basics and Intermediate

Learning Outcomes

Upon completing your learning, you will be able to

  • Differentiate between Server Side Rendering (SSR) and Client Side Rendering (CSR) and grasp the benefits and drawbacks of each approach.
  • Explore the concept of Embedded JavaScript (EJS) as a powerful templating language for generating dynamic HTML content.
  • Learn how to set up a Server Side Rendering (SSR) back-end application that renders content on the server before sending it to the client.
  • Dive into EJS templating syntax and understand how to integrate dynamic data into HTML templates using EJS tags.
  • Discover the process of creating front-end views using EJS, combining HTML and CSS to craft visually appealing user interfaces.
  • Explore the role of EJS Middleware in handling and rendering ".ejs" templates, enabling seamless integration with your Express.js application.

Tutorial Steps

Total steps: 7

  • Step 1: Setting Up Our Workspace

    First, open your coding workspace of your choice. In our case, we will be using VS Code. Open your terminal in your code editor and ensure that the current working directory is the root folder (StackUp-BackEnd-ExpressJS).

    If the current working directory does not reflect the root folder, you can use the cd command followed by the path to your folder.

    Next, in the code editor, open up the terminal and enter the following commands.

    mkdir "Views-with-EJS"
cd "Views-with-EJS"
npm install express morgan body-parser ejs

    In the code above, there are 3 commands. First, using “mkdir”, it creates a directory within our root directory. The next command will change the current working Directory to the new folder we have created. Lastly, the last command will install the following 4 packages: Express, Morgan, EJS and Body Parser. These will be the only dependencies needed.

    Lastly, an optional package that is highly recommended is to use the “nodemon” package. This package is helpful in helping you to reload the server application whenever a change is detected. It is like the local development server in our React campaigns. You can obtain it by entering the following command: npm install nodemon -g --save

    The “-g” is known as a flag. Flags are usually optional and not needed. However, there are additional commands to be added to the current command function. In this case, “-g” for node means “Global”. Any packages installed in this will be stored in a Global directory on your local machine. This means that any project that requires this package can refer to this package without having to install it in the future.

  • Step 2: Public Folder

    In Express, we will need a place to store our files that can be accessible to the public. For example, CSS Files, Images, Videos and more. In our project folder (Views-with-EJS), create a new folder called public.

    Further into this quest, we will be using this folder to store any files for users.

  • Step 3: app.js

    We will now create the entry file for the back-end application. In Views-with-EJS, create a new file, app.js. Once you have created the file, open it and add the following code as shown below. 

    const express = require('express');
const morgan = require('morgan');
const bodyParser = require('body-parser');

const app = express();
const port = 3000;

// Configure EJS as the view engine
app.set('view engine', 'ejs');

// Middleware
app.use(morgan('dev')); // Logging middleware
app.use(bodyParser.json()); // JSON parsing middleware

// Serve static files from the 'public' directory
app.use(express.static('public'));

// Define routes
app.get('/', (req, res) => {
res.render('index',{
	time: new Date().toString()
});
});

app.get('/about', (req, res) => {
    res.render('about');
});

// Start the server
app.listen(port, () => {
    console.log(`Server is running on http://localhost:${port}`);
});
    JavaScript

    We begin by importing the necessary modules. The express module in line 1 is crucial for creating an Express app, while morgan in line 2 is used for logging HTTP requests, and body-parser in line 3 is essential for parsing incoming JSON data.

    Here, we create an instance of the Express app using the express() function in line 5. Additionally, we establish a constant named port in line 6, and this constant is set to 3000, signifying the port where our server will listen for incoming requests.

    The next step is to configure EJS as the view engine for rendering templates. In line 9, we use app.set() to set view engine to ejs. This configuration prepares Express to render EJS templates.

    As we proceed, we integrate middleware functions into the Express app. Line 12 employs morgan('dev') to apply logging middleware, which will log incoming requests and their details to the console. Line 13 then utilizes bodyParser.json() to enable the parsing of incoming JSON data in the request body.

    Static files, such as images, stylesheets, and JavaScript files, are typically served from a "public" directory. In line 16, we use the express.static('public') middleware to serve static files from the 'public' directory.

    Defining routes is an essential aspect of web applications. Here, we create routes using the app.get() method. In line 19, a route is set up for the root URL ('/'), and when a client sends a GET request to this URL, the server responds by rendering the 'index' EJS template. Similarly, in line 25, a route is established for the '/about' URL, and it renders the 'about' EJS template when accessed.

    Finally, we initiate the server to start listening on the designated port. In line 30, we use app.listen() with the specified port, and a callback function logs a message to the console, indicating that the server is operational. The provided URL http://localhost:${port} is where the server can be accessed locally.

    In essence, this code snippet showcases the foundational setup of an Express.js server with EJS templating. It configures EJS as the view engine, integrates middleware functions for logging and JSON parsing, serves static files from the 'public' directory, defines routes for rendering EJS templates, and initiates the server for listening.

  • Step 4: Creating Views

    We will now learn how we can create templates for EJS. EJS is a templating engine that contains HTML and CSS elements. However, it will also contain the special syntax that will also allow ExpressJS to render a final copy of the HTML file to be sent to the Front End. This is extremely helpful in websites that require dynamic content. This means that we are also allowed to make reusable templates like a Website Header, Navigation Bar, to be displayed on various pages throughout the platform.

    In the “Views-with-EJS” folder, create a new folder called views. Within the views folder, create a new file called about.ejs. Open the file and add the code below.

    <!DOCTYPE html>
<html>
<head>
    <title>About Us</title>
    <link rel="stylesheet" type="text/css" href="/styles.css">
</head>
<body>
    <header>
        <div class="container">
            <h1>About Me</h1>
        </div>
    </header>

    <main>
        <div class="container">
            <p>Lorem ipsum sample text</p>
            <a href="/">Go back to the home page</a>
        </div>
    </main>

    <footer>
        <div class="container">
            &copy; Stackup Introduction to Back End Programming with NodeJS
        </div>
    </footer>
</body>
</html>
    HTML

    At the moment, this is a typical HTML file and you can treat it as such. However, there is much more we can do using EJS. EJS allows us to dynamically add and modify the contents of this file programmatically. In line 5, we have also added a reference to the stylesheet to be applied to this view. The reference is in relation to the public folder that we have created in Step 2. 

    ⚠️ Important: Do note that in line 16, within the paragraph tags (<p></p>), replace 'Lorem ipsum sample text' with a one to two sentence introduction about yourself. Be sure to include your Campus App username in the introduction. 

    Here is how you can create a new view using EJS. Within the views folder, create a file called index.ejs and add the following code as shown below within the file.

    <!DOCTYPE html>
<html>
<head>
    <title>Express EJS Project</title>
    <link rel="stylesheet" type="text/css" href="/styles.css">
</head>
<body>
    <header>
        <div class="container">
            <h1>Welcome to Stackie's website!</h1>
        </div>
    </header>

    <main>
        <div class="container">
            <p>This is the home page.</p>
            <a href="/about" class="button">Learn more about us</a>
            <p>The current time is <%= time %></p>
        </div>
    </main>

    <footer>
        <div class="container">
            &copy; Stackup Introduction to Back End Programming with NodeJS
        </div>
    </footer>
</body>
</html>
    HTML

    This file will be the index file that will be rendered and sent to the client to display on the web browser. Do take note of line 18 where we have added a way where we can dynamically input the variable into this file without having to manually edit the file. We will use the symbols <%= %> to output the variable.

    Let us open the app.js file. In line 21, you may notice that we have added an object as the second argument for the res.render() method. Within the object, we have also parsed a Key-Value Pair, where the key is the variable within the EJS file, and the value is the value that is expected to be displayed on our client’s web browser. This is how we can dynamically input a value into the EJS file. In this case, the key is “time”.

  • Step 6: Running our Server Script

    We will now be running the script, app.js. In our terminal, ensure that our Current Working Directory (CWD) is Views-with-EJS, the folder to which we have added our packages, and app.js.

    Then, use the command node app.js to kick-start your server application.

    For those who have installed “nodemon” previously, you may use nodemon app.js. The advantage of nodemon is that it will automatically reload the server once it has detected any changes within the CWD. In this case, once there is a code change within the “Views-with-EJS” folder, the server will reload. If you would like to use nodemon but have yet to install it, please refer to step 4.

    We have just created our very own server which will serve the Front End components to the Front End. Now, in your internet browser, enter "http://localhost:3000" in your browser.

    For Gitpod users, you should see a pop up appear at the bottom right corner of your browser. Click on 'Open Browser'. If the pop up does not appear, go to Ports and click on the address for port 3000.

    Your browser should show the following output as shown below.

    This is working because as we enter the url, “http://localhost:3000”, our browser will make a GET request to our server and right after, our server will respond to the GET request (line 19 of app.js). For this instance, the server will respond to the request by rendering the “index” EJS component onto the Front End.

    Once we have clicked on the “Learn more about us” button, it will make a new request to the server. It will be a “GET /about” request and our server will respond to the request in line 25 of app.js and it will render the “about” EJS component. Your webpage will then show an "About Me" page.

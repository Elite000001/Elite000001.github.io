A Quick Guide to Firebase

4 steps

This tutorial provides an express walkthrough of installing and using Firebase and will allow you to be caught up to speed before you attempt web development quests that require deploying your site. You are still advised to complete the other two tutorials of the Firebase module to get a good understanding of Firebase. In the event you face any issues with initialising or deploying to Firebase, check out Step 4 for troubleshooting.
Join us on Discord

Learning Outcomes

At the end of this tutorial, users will be able to

  1. Create projects on the Firebase console
  2. Run commands using the Firebase CLI tools
  3. Deploy files from their local workspace to Firebase
  4. Resolve errors they encounter when deploying files on Firebase

Tutorial Steps

Total steps: 4

  • Step 1: Installations and Account Creation

    To use Firebase, you will first need to have a Google account. Head to the Firebase platform and click on the "Go to Console" button at the top-right hand corner of the page.

    Next, we will need to install NodeJS. Download a suitable installer and run the installer.

    Finally, to install the command line interface (CLI) tool for Firebase. We will be using the CLI to upload and deploy files for hosting. In your terminal, enter the following command.

    npm install -g firebase-tools

    To check if the installation was successful, run the command below. If a version number is returned, your installation was successful.

    firebase -V

    Some users may face privileged restrictions or errors. This is due to Node installing Firebase into a folder which requires administrator privileges. This can be solved by doing the following:

    For Window users:

    Open command prompt as an administrator. Then run the necessary commands for installation.

    For MacOS and Linux users:

    Type the following command into your terminal.

    sudo firebase -V
  • Step 2: Initiating a Project on Firebase

    At this stage, it is presumed that you have already completed your website and are ready to upload your files to Firebase. For any troubleshooting issues, see Step 4.

    1. In your Firebase Console, click on Add project.
    2. Give your project a name. Do take note of the naming conventions for project names. It must be 6 to 30 lowercase letters, digits or hyphens. It must start with a letter. In the event that the project name is already taken, you can add some numbers to the end of the name to make it unique.
    3. You will be asked if you want to enable Google Analytics for your Firebase project. This is optional for us now as we do not need to track site metrics. 
    4. Then, proceed to create project.
    5. In your file directory, ensure you have a build folder present with the files that you want to upload. This could have been created when you ran the npm run build command (assuming you are working with React). If the folder is not created, create it.
    6. In your terminal where you are in the app directory (e.g. counter-app or blog-site), run the firebase init command.
    7. When asked which Firebase feature to set up for the directory, use your arrow keys to navigate down and then use the spacebar to select "Hosting: Configure files for Firebase Hosting and (optionally) set up GitHub Action deploys". Then press enter. 
    8. When asked which project directory to associate with a Firebase project, select Use an existing project. Select the name of the project you created earlier. 
    9. When asked what do you want to use as your public directory, indicate build. 
    10. For the following questions on configuring as a single-page app, set up automatic builds, and overwriting of index.html, indicate 'n' for all of them. 

    If your Firebase initialisation is successful, you should see the message "Firebase initialization complete!"

  • Step 3: Deploying of Files

    At this point, ensure all the files and folders you wish to deploy to Firebase are in the build folder.

    In your terminal, enter the command

    firebase deploy

    Your terminal output should be similar to what you see below if your deployment is successful.

  • Step 4: Troubleshooting

    Before checking out the troubleshooting solutions, do check that you have

    1. Understood and followed the quest instructions correctly

    2. Ran the firebase init and deploy commands in the root directory of your project

    .

    1. I received an error message of "Failed to create project. See firebase-debug.log for more info". when naming my project in the CLI.

    Ideally you should have created your project on Firebase console instead of the CLI.

    In the directory you were running the firebase command, open the firebase-debug.log file. Try to read for a specific error. If the specific error says "Callers must accept Terms of Service", visit https://console.firebase.google.com, then click 'add project' and follow the steps to create the project. Once that is done, when initiating Firebase, instead of creating a new project, you should select the option 'use existing project'.

    If the specific error says "HTTP Error: 403, The caller does not have permission", you may want to check that you are logged in to Firebase on your terminal. Run the command firebase logout and then firebase login.

    2. When running commands in Windows Powershell, the error message is "firebase.ps1 cannot be loaded because running scripts is disabled on this system".

    For Windows users, it is recommended to use command prompt or Git Bash to run the commands instead.

    However, if you wish to continue with Windows Powershell, run the command Set-ExecutionPolicy RemoteSigned -Scope CurrentUser. Then continue with the Firebase commands.

    3. When I visit my hosting URL, I see a page that says "Firebase Hosting Setup Complete". Where is the website I built?

    You are likely seeing this because the index.html file in your build directory was overwritten with Firebase code. This happens if you chose Y in the command line interface when the file was asked if you wish to overwrite index.html.

    Replace the files in your build directory with updated files and proceed to redeploy using the firebase deploy command in your command line interface.

    4. When I visit my hosting URL, I see a page that says "404 Page Not Found".

    When asked to indicate your public directory, you did not indicate the build folder. As a result, the files in your build folder were not deployed to Firebase. Start from the beginning by running firebase init and ensure that you follow the instructions carefully. If you feel that the hosting URL is not working, you can run the command firebase deploy --only hosting

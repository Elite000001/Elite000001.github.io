Tutorial 1 - Introduction to Taipy

8 steps

With the demand for interactive and dynamic front-end interfaces, you'll find yourself comfortable with Taipy. Taipy is an open-source Python library designed to simplify the creation of both front-end and back-end features for web applications. Here, you will dive into Taipys Visual Components, learning to create engaging and interactive dashboards that cater to a wide range of data visualization needs. From tables and charts to dynamic inputs and selectors, youll explore how Taipys GUI elements simplify building sophisticated interfaces. You will be integrating and customizing these components to deliver seamless user experiences while leveraging Taipys intuitive syntax. By the end, youll have the skills to design and deploy functional dashboards that can adapt to real-time data and user interactions.

Tutorial Steps

Total steps: 8

  • Step 1: Understanding Taipy

    Taipy is an open-source Python library maintained by Avaiga, designed to simplify the creation of interactive, data-driven web applications. Look at Taipy Gallery to see what others have built using Taipy, it provides a seamless interface between data processing and visual representation, allowing developers to focus on logic while minimizing boilerplate code. With Taipy, you can quickly build dynamic and responsive applications.

    Key Features of Taipy:

    Python-Centric: Create complete web applications directly in Python. Taipy seamlessly works with Pandas, NumPy, Plotly, and other data science and machine learning libraries, making it ideal for building interactive dashboards and tools.

    Rich UI Components: Offers pre-built reactive components like charts, tables, sliders, and more, directly linked to Python variables.

    Backend Powered by Flask: Taipy uses Flask for robust, scalable backend operations.

    Multi-User Support: Taipys state management ensures independent sessions for every user.

  • Step 2: Setting Up the Environment

    To begin, create a new folder named TaipyTrail. Open this folder in your preferred Integrated Development Environment (IDE). We are using Visual Studio Code (VSCode) for this tutorial, but feel free to use any IDE you are comfortable with.

    Next, ensure your terminal is set to the project's root directory (the TaipyTrail folder you just created). All subsequent commands should be run from this directory. After that, create a Python virtual environment to manage the projects dependencies.

    Run the following command in your terminal:

    python3 -m venv .venv

    Note: You might wonder why python3 was used instead of just python. This is because, on some systems, python may still refer to Python 2, while python3 explicitly calls Python 3, which is what this project needs. Depending on your system setup, python could point to Python 3, but its good practice to specify python3 to avoid any ambiguity, especially when working in environments where both versions may be installed.

    Now, activate the virtual environment to ensure all dependencies are installed and run within it. Use the appropriate command for your operating system:

    On macOS/ Linux:

    source .venv/bin/activate

    On Windows:

    .venv\Scripts\activate

    Once activated, your terminal prompt should change to include (.venv), indicating the virtual environment is active. For example:

    (.venv) your-computer-name:TaipyTrail username$

    Finally, let's install Taipy. Do ensure you have Python 3.9+ installed. You can check your version of python by using the following command: 

    python --version

    For installing Taipy, use the following command:

    pip install taipy==4.0.1

    Next, run the following command to check if Taipy is successfully installed:

    pip show taipy

    Expected Outcome:

    The command should display details about the installed package, such as the version and location. For example:

    Name: taipy
Version: 4.0.1
Summary: A 360 open-source platform from Python pilots to production-ready web apps.
Location: /path/to/.venv/lib/python3.X/site-packages
  • Step 3: Basics of Taipy

    Now, lets create a simple application with Taipy and then we will explore these components one by one.

    In the root directory of your project, create a new file named main.py, and add the following code:

    from taipy import Gui

if __name__ == "__main__":
    Gui(page="# Hello, *Taipy*!").run(title="Hello, Taipy")
    Python

    Run the following command to run our application:

    taipy run main.py

    Now visit http://127.0.0.1:5000 to check our first application on Taipy, You should see the text Hello, Taipy! displayed on the webpage. This text is rendered in Markdown format, as defined in the Gui(page="...") argument and if you hover over the tab you will see "Hello, Taipy!" as the title, 5000 is the default port for Taipy applications.

    Now lets understand the code, we are importing the Gui class from Taipy and using its run() method to start the server that delivers pages to web clients. The Gui class in Taipy is the core object that connects your Python backend to the user interface. It takes your page definitions, whether in Markdown, Python, or HTML, and turns them into interactive web applications.

    Note: Taipy supports multiple definitions, such as markdown, HTML, and Python API, but we will work with Python API for this campaign.

    Taipy provides two important optional command line arguments to enhance your development process: debug, and use-reloader.

    debug: This option enables debug mode, which provides detailed stack traces of any errors within the application. If an error occurs, the terminal will display a traceback message, helping you quickly identify and resolve issues.

    use-reloader: By default, the application does not automatically reload when you modify the code. Enabling this option ensures the application reloads automatically whenever you save changes to a file.

    To use these features, run the following command:

    taipy run main.py --debug --use-reloader
  • Step 4: Exploring Basic Taipy Components

    In this step, we will explore some of the basic components of Taipy, such as text, input, number, and more.

    Every visual element in Taipy follows a similar syntax:

    import taipy.gui.builder as tgb

tgb.visual_element_name("{variable}", param_1=value_1, param_2=value_2, ...)
    Python

    Text Component Example

    In the previous step, you ran a command to have your server running. If the server is not running, start it using the command:

    taipy run main.py --debug --use-reloader

    Replace the code in main.py with the following:

    from taipy import Gui
import taipy.gui.builder as tgb

text_value = "# Hello, *Taipy*!"

with tgb.Page() as page:
    tgb.text("{text_value}", mode="md", hover_text="Static Text") # mode: md/markdown, pre, raw

if __name__ == "__main__":
    Gui(page).run(debug=True)
    Python

    Save the file and head over to your preferred browser to see our application in action. , now lets understand the code. First, we declared a variable text_value, which contains a Markdown string (# Hello, *Taipy*!). This string will be rendered as a formatted header, with # indicating a header and *...* italicizing the word "Taipy." Next, we used tgb.Page() as a context to define the structure of our UI. The tgb.text method is used to create a text component within this context.

    Note: We are only covering some of the component's properties. For a complete list, refer to the Taipy Components Documentation.

    Moving on lets see other components in action and understand the concept of state in Taipy components:

    Taipy uses a state object to manage variables within your application. The state object holds all variables for each user session, enabling independent multi-user interactions. When a UI component's bound variable changes, the state object is updated, enabling dynamic and reactive behavior. For example, if a user modifies the value in an input field, the state object reflects this change immediately, enabling dynamic and reactive behavior in your application.

    The on_change() Function:

    The on_change() function is a special callback in Taipy. It is automatically triggered whenever a variable's value is updated. Taipy looks for this function in your application and invokes it with the following three parameters:

    state: The state object containing all the variables.

    var_name: The name of the variable that was modified.

    var_value: The new value of the modified variable.

    This function allows you to add custom logic to respond dynamically to changes in variables. For example, you can reset variables, trigger actions, or update other components based on the new value.

    Time to see it in action, replace the code in main.py with the following:

    import taipy.gui.builder as tgb
from taipy.gui import Gui, notify

# Callback for button actions
def on_button_action(state):
    notify(state, 'info', f'The text is: {state.text}')
    state.text = "Button Pressed"

# Special callback for state changes
def on_change(state, var_name, var_value):
    if var_name == "text" and var_value == "Reset":
        state.text = ""
    elif var_name == "slider_value":
        state.number_value = var_value * 2  # Update number_value based on slider
    elif var_name == "toggle_state":
        notify(state, 'info', f'Toggle is now {"ON" if var_value else "OFF"}')

if __name__ == "__main__":
    # Initial state variables
    text = "Original text"
    slider_value = 50
    number_value = 100
    toggle_state = True
    selected_date = None
    uploaded_file = None

    # Define the UI page
    with tgb.Page() as page:
        tgb.text("# Taipy Input Components", mode="md")
        tgb.text("My text: {text}")
        
        # Input component
        tgb.input("{text}", label="Enter your text")
        
        # Slider component
        tgb.slider("{slider_value}", min=0, max=100, step=5, label="Adjust slider")
        
        # Number component
        tgb.number("{number_value}", label="Number Input")
        
        # Toggle component
        tgb.toggle("{toggle_state}", label="Toggle state")
        
        # Date picker component
        tgb.date("{selected_date}", label="Pick a date")
        
        # File upload component
        tgb.file_selector("{uploaded_file}", label="Upload a file", extensions=".csv,.xlsx")

        # Button component
        tgb.button("Submit", on_action=on_button_action)

    # Run the application
    Gui(page).run(debug=True)
    Python

    When you refresh the page, you'll see an interactive interface where you can explore various components in action. Try typing into the text input field to see the displayed text update instantly, or adjust the slider to notice how the number input changes dynamically. Toggle the switch to observe notifications about its state, and experiment with the date picker or file upload to see how they interact with the application state. Finally, click the "Submit" button to trigger a notification and update the text. This allows you to experience real-time state management in a user-friendly way.

    The state object manages the text variable and dynamically links to the tgb.text and tgb.input components, where changes to the input field automatically update state.text. The slider_value is connected to a slider, and its changes dynamically update the number_value to twice the slider's value through the on_change function. Similarly, the toggle_state variable reflects the toggles position, with the on_change function triggering a notification whenever the toggle is switched. Clicking the button invokes the on_button_action function, displaying a notification with the current value of state.text and updating it to "Button Pressed." This seamless interaction between state management and callbacks ensures real-time updates and dynamic behavior across all components.

    You may be wondering how our .css file was linked to our application. The default value for this parameter is a file located next to your main Python script, with the same name, except for the extension, which must be .css, which in our case was main.css. If you have a different file name for your .css file, you can specify it in the css_file parameter of the Gui constructor.

  • Step 6: Layouts in Taipy

    Taipy provides a flexible way to organize components into structured layouts. Using layout components like layout, part, and expandable, you can create visually appealing and intuitive designs for your application. Let's explore these components step by step.

    Replace the content of main.py with the following template:

    from taipy.gui import Gui
import taipy.gui.builder as tgb

with tgb.Page() as page:
    # We will add components here one by one

Gui(page).run(debug=True)
    Python

    The expandable component creates collapsible sections to organize content. Add the following code at line 6:

        # Expandable: Creates a collapsible section to group content
    with tgb.expandable("Expandable Section"):
        tgb.text("This is an expandable section. Click to open or close.")
    Python

    Expected Outcome: A collapsible section titled "Expandable Section" with text inside.

    The part component groups multiple elements together and can dynamically show or hide them. Add the following code at line 9:

        # Part: Groups elements and can be conditionally displayed
    with tgb.part(render="{True}"):  # Use render="{False}" to hide this part
        tgb.text("This text is inside a part.")
        tgb.button("Click Me")
    Python

    Expected Outcome: A visible section with text and a button. You can toggle visibility by changing render.

    If we change render to False, the part component will disappear:

    The layout component arranges elements in columns or rows using the columns property. Add the following code at line 13:

        # Layout: Arranges elements in a column-based layout, 2:1 ratio
    with tgb.layout(columns="2 1"):
        with tgb.part():
            tgb.text("Column 1")
        with tgb.part():
            tgb.text("Column 2")
            tgb.text("some text here")
    Python

    Expected Outcome: Two columns with text labeled "Column 1" and "Column 2" with their widths in a 2:1 ratio.

    Combining Layouts:

    You can combine these layouts to create complex structures. Add the following code at line 20:

        # Combining layouts
    with tgb.layout(columns="1 2"):
        with tgb.part():
            tgb.text("Column 1: Nested Layout")
        with tgb.expandable("Column 2: Expandable"):
            tgb.text("This expandable is inside a column layout.")
    Python

    Expected Outcome: A section with text labeled "Column 1: Nested Layout.", and an expandable section labeled "Column 2: Expandable" with text inside.

  • Step 7: Layouts in Taipy II

    Dialogs allow you to present content in a modal window over the main application page. Users can interact with the dialog, which can be closed via buttons or by clicking outside the dialog area.

    Replace the content of main.py with the following code:

    import taipy.gui.builder as tgb
from taipy.gui import Gui

show_dialog = False

def dialog_action(state, _, payload):
    if payload["args"][0] == 0:
        print("Good to hear!")
    elif payload["args"][0] == 1:
        print("Sorry to hear that.")
    else:
        print("Ok bye.")
    state.show_dialog = False

with tgb.Page() as page:
    with tgb.dialog("{show_dialog}", title="Welcome!", on_action=dialog_action, labels="Couldn't be better;Not my day"):
        tgb.html("h2", "Hello!")

    tgb.button("Show", on_action=lambda s: s.assign("show_dialog", True))

if __name__ == "__main__":
    Gui(page).run(title="Dialog - Labels")
    Python

    Expected Outcome: Clicking "Show" opens a modal dialog titled "Welcome!", Inside the dialog: A header with the text "Hello!" and Two buttons: "Couldn't be better" and "Not my day."

    Taipy makes it easy to create multi-page applications with navigation. Follow these steps to define multiple pages.

    Replace main.py with the following code:

    from taipy.gui import Gui
import taipy.gui.builder as tgb

# Define the root page with navigation
with tgb.Page() as root_page:
    tgb.navbar()  # Default navigation bar
    tgb.text("# Welcome to the Multi-Page Application", mode="md")

# Define Page 1
with tgb.Page() as page_1:
    tgb.text("## This is Page 1", mode="md")

# Define Page 2
with tgb.Page() as page_2:
    tgb.text("## This is Page 2", mode="md")

# Combine all pages
pages = {
    "/": root_page,
    "page1": page_1,
    "page2": page_2
}

# Run the application
Gui(pages=pages).run()
    Python

    Expected Outcome: A multi-page application with a navigation bar at the top, showing two tabs: "Page1" and "Page2". Try navigating between these 2 pages, and see how easy it is to create multi-page applications using Taipy.

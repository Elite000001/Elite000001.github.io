Styling a Card Component

5 steps

In this tutorial, you will be styling card components. Cards are trending these days. They are one of the most popular components of a web page because they allow you to present information in an interesting and organized way.

The card that you will be creating in this tutorial has a more elaborate design than you are probably used to creating. You will learn some CSS properties that will allow you to build upon your existing skillsets.
Join us on Discord

Learning Outcomes

By the end of this tutorial, you will be able to:

  1. Use background border and box-shadow properties and values.
  2. Implement filters in and behind elements using filter and backdrop-filter properties.
  3. Avoid code repetition using CSS Variables.

Tutorial Steps

Total steps: 5

  • Step 1: Background and Borders

    Background and border seem to be pretty simple properties with just solid colors, but they are actually very interesting properties. In this step you will read the MDN documentation about these properties and understand the usefulness of background and border properties.

    Take note that you can use this documentation as reference material during your development process too!

    🔗 Backgrounds and borders

  • Step 2: Filters in CSS

    In CSS you can apply filters in and behind elements. These filters are often used to create both more interesting designs and improve the UX of the web page. In this step you will read an article that will show you all the values of the filter and backdrop-filter properties.

    While reading, why not take note of some of the values that you might want to consider using in your future website designs?

    🔗 Filters

  • Step 3: CSS Variables

    CSS variables or custom properties is a fairly new feature in CSS, but also very useful for reducing repetition in CSS. It can also be used to create powerful runtime effects like theme switching. You can assign different values to CSS variables, and we will be watching a tutorial to learn what values can be used. Don't forget to make your own notes or a cheat sheet on this so you can quickly refer to them while coding!

  • Step 4: Exercise: Styling a Card Component

    To put what you have learnt to the test, you are required to style a trending card component, just like the one you see in 'expected output'.

    • Open up a code editor of your choice.
    • Next, head over to this GitHub repository. Copy the code from index.html and style.css into your HTML and CSS files.
    • For this task, you will need to add some CSS properties and values. You will know where to add them by looking out for /* ADD YOUR CODE HERE */. You are strongly advised to study the classes in the HTML code to understand how your CSS code will affect the web page. Here are also some hints of how the card component works:
      • This card component has important values as CSS Variables, so make sure to use all of them.
      • Do note that the card and the icons under 'My Technologies' have top and left side borders.
      • Replace “StackUp Username” in the heading text with your own StackUp username.
      • For the different properties, experiment with different values and see what works.
  • Step 5: Checking Your Work

    You've spent time styling your own web page. It's now time to check your work. Use the following pointers to help you check your work:

    • Did you make use of the CSS variables for your background image, such as var(--body-bg-image)? 
    • Did you make use of the Tech Icon Background colour variable (--tech-icon-bg-clr) when styling the icon background?

    Finally, you can check your CSS code against this suggested solution.
